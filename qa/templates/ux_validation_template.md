## 🎨 UX 검증 체크리스트

### 1. UI 일관성 (Visual Consistency)
- [ ] 색상 팔레트가 디자인 가이드라인과 일치하는가?
- [ ] 폰트 크기 및 굵기가 일관적인가?
- [ ] 버튼 스타일 (크기, 모서리 반경, 그림자)이 통일되어 있는가?
- [ ] 간격 (padding, margin)이 일관적인가?
- [ ] 아이콘 크기 및 스타일이 통일되어 있는가?
- [ ] 색상 테마 (Primary, Secondary, Accent)가 명확한가?

### 2. 반응형 웹 (Responsive Design)
- [ ] 모바일 (< 768px)에서 레이아웃이 깨지지 않는가?
- [ ] 태블릿 (768px - 1024px)에서 정상 작동하는가?
- [ ] 데스크톱 (> 1024px)에서 여백이 적절한가?
- [ ] 모바일에서 터치 타겟이 44x44px 이상인가?
- [ ] 가로/세로 모드 (orientation) 전환 시 깨지지 않는가?
- [ ] 모바일에서 가로 스크롤이 발생하지 않는가?

### 3. 접근성 (Accessibility / A11y)
- [ ] 키보드만으로 모든 기능에 접근 가능한가? (Tab, Enter, Esc)
- [ ] 포커스 상태 (focus)가 시각적으로 명확한가?
- [ ] ARIA 레이블 (aria-label, aria-labelledby)이 있는가?
- [ ] 색상 대비가 WCAG 기준 (4.5:1)을 만족하는가?
- [ ] 스크린 리더 (Screen Reader)로 읽을 수 있는가?
- [ ] 이미지에 대체 텍스트 (alt)가 있는가?
- [ ] 폼 입력 필드에 label이 연결되어 있는가?

### 4. 사용자 피드백 (User Feedback)
- [ ] 로딩 중 스피너 또는 "처리 중..." 메시지가 있는가?
- [ ] 에러 메시지가 사용자 친화적인 한글인가?
- [ ] 성공 메시지 (Toast, Modal)가 명확한가?
- [ ] 에러 발생 시 다음 액션 (재시도, 취소)이 제공되는가?
- [ ] Toast 메시지 표시 시간이 적절한가? (3-5초)
- [ ] Modal 창에 닫기 버튼 (X)이 있는가?
- [ ] 빈 상태 (Empty State) 안내 메시지가 있는가?

### 5. 인터랙션 (Interaction)
- [ ] 버튼에 :hover, :active, :focus 상태가 정의되어 있는가?
- [ ] 클릭 시 시각적 피드백 (ripple, scale)이 있는가?
- [ ] 애니메이션 (transition, animation)이 자연스러운가?
- [ ] 드래그 앤 드롭이 직관적인가?
- [ ] 스크롤 애니메이션 (fade-in, slide-in)이 과하지 않은가?
- [ ] 로딩 시간이 3초 이상일 때 진행률 표시가 있는가?
- [ ] 더블 클릭 방지 (debounce)가 적용되어 있는가?

### 6. 다크모드 (Dark Mode)
- [ ] 다크모드에서 텍스트 가독성이 유지되는가?
- [ ] 다크모드에서 색상 대비가 충분한가?
- [ ] 다크모드 전환 시 애니메이션이 부드러운가?
- [ ] 다크모드에서 이미지/아이콘이 보이지 않는 문제가 없는가?
- [ ] 다크모드 설정이 localStorage에 저장되는가?

### 7. 성능 및 로딩 (Performance)
- [ ] 페이지 로딩 시간이 3초 이내인가?
- [ ] 이미지가 최적화되어 있는가? (WebP, lazy loading)
- [ ] CSS/JS 파일이 압축되어 있는가?
- [ ] 불필요한 리렌더링이 없는가?
- [ ] 무한 스크롤이 있는 경우 가상화 (virtualization)가 적용되어 있는가?
