## 1. 테스트 목적
이번 v3.1 업데이트에서 추가된 **전용 모달 기반 프로젝트 등록, 업무 수락 워크플로우, 슬랙 알림(3회
), 참고사항 히스토리(CRUD)** 기능이 사용자 요구사항 및 자비스 팀의 실행 계획(task.md)대로 정상 작동하는지 수동으로 확인합니다.

## 2. 테스트 환경
- **플랫폼**: 주디 워크스페이스 웹 사이드바
- **권한**: 
  - 사용자 A (업무 요청자)
  - 사용자 B (업무 담당자)
  - 관리자 (용남 대표님 등)

## 3. 세부 테스트 케이스 (Test Cases)

### [TC-01] 프로젝트 직접 등록 (인라인 UI)
| 스텝 | 수행 내용 | 기대 결과 |
| :--- | :--- | :--- |
| 1 | 업무 등록 모달 진입 | 기존 프로젝트 목록이 정상 로드됨을 확인 |
| 2 | `+ 새 프로젝트` 버튼 클릭 | 인라인 입력창이 토글되며 나타남 |
| 3 | 프로젝트명 입력 후 `생성` 클릭 | 상단에 성공 토스트 노출 및 드롭다운에 즉시 반영 |
| 4 | (검증) 구글 시트 | `Projects` 시트에 새 프로젝트와 자동 생성된 코드가 기록됨 |

### [TC-02] 업무 등록 및 담당자 지정 (수락 워크플로우)
| 스텝 | 수행 내용 | 기대 결과 |
| :--- | :--- | :--- |
| 1 | 업무 등록 시 타인(사용자 B)을 담당자로 지정 | `수락대기` 상태로 자동 설정됨을 확인 |
| 2 | 시작예정일 및 마감일 지정 | 시트에 J열(시작일), K열(등록자)에 정상 기록됨 |
| 3 | (검증) 사용자 B의 대시보드 | `내 업무` 리스트 상단에 [✅ 수락] 버튼이 활성화됨 |
| 4 | 사용자 B가 `수락` 버튼 클릭 | 상태가 `진행중`으로 변경되며 요청자(사용자 A)에게 슬랙 DM 전송 |

### [TC-03] 슬랙 알림 (DM 발송 규칙)
| 스텝 | 수행 내용 | 기대 결과 |
| :--- | :--- | :--- |
| 1 | 시작일이 D-1, D-day인 업무 생성 | 트리거 실행 시 담당자에게 DM이 즉시 발송되는지 로그 확인 |
| 2 | 시작일 D-3 알림 대기 | 매일 오전 9시 정기 트리거에서 대상자에게만 DM 발송 확인 |

### [TC-04] 참고사항(구 메모) 히스토리 및 자동 링크
| 스텝 | 수행 내용 | 기대 결과 |
| :--- | :--- | :--- |
| 1 | 업무 상세 보기 하단 `+ 추가` 클릭 | 참고사항 입력창 오픈 및 텍스트/링크 입력 |
| 2 | 링크(https://...) 포함하여 저장 | 타임라인에 렌더링되며 클릭 시 새 탭으로 즉시 이동(매직 링크) |
| 3 | 본인 작성 글 `✏️` 수정 | 원본 텍스트가 로드되고 수정 후 저장 시 타임라인 즉시 업데이트 |
