## 지시 원문

> 소환된 Agent 팀에 없는 능력, 팀원, 기술이 기존 프로젝트 또는 새로운 프로젝트에 요구사항일 경우 팀에서 어떻게 대응할 수 있는지에 대한 가이드가 필요해.
> 예시: 자비스팀 소환 → 기술 역량 GAS 한정 → 개발 요구내용 파악해보니 파이썬, 자바 필요 → 대응 방법 시나리오가 없음.
> 실제 에이전트 팀원이 역량을 확장해서 갭확인 대안 제시 실행 하는 프로세스. 핵심은 다른 에이전트팀에게 전달할 위임의 업무를 확장해서 하는게 아니라, 자기 확장.

## 팀장 이해 요약

- **핵심 요청**: "역할은 내 소관인데 기술이 부족한 경우" 에이전트가 역량을 확장하여 대응하는 프로토콜 신설
- **작업 범위**: COMMON_RULES.md에 규칙 6(capability_gap) 추가, 5개 팀 DESIGN.md 인라인 블록 21개소 업데이트
- **완료 기준**: 모든 에이전트가 역량 갭 발견 시 "갭 보고 → 확장 제안 → 승인 → 확장 실행" 프로세스를 따름
