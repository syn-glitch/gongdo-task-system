## 📝 문서 작성 원칙

### 1. 파일명 규칙

**템플릿**:
```
TEMPLATE_[유형].md
```

**일반 문서**:
```
YYYY-MM-DD_[제목].md
```

**가이드**:
```
[주제명].md (소문자, 하이픈 연결)
```

예시:
- `TEMPLATE_feature_spec.md`
- `2026-02-28_folder_restructure_guide.md`
- `naming-convention.md`

### 2. 마크다운 스타일

- **제목 체계**: `#` (H1) → `##` (H2) → `###` (H3), 최대 4단계
- **강조**: `**굵게**`, `*기울임*`, `~~취소선~~`
- **코드**: 인라인 `` `코드` ``, 블록 ` ```언어 ... ``` `
- **링크**: `[텍스트](경로)`
- **리스트**: `-` (비정렬), `1.` (정렬)

### 3. 메타 정보 블록

모든 문서 상단에 메타 정보를 포함합니다.

```markdown
**문서 버전**: v1.0
**작성일**: YYYY-MM-DD
**작성자**: [이름 (역할)]
**최종 수정**: YYYY-MM-DD
**검토자**: [이름]
```

## 💡 문서 작성 팁

### 좋은 문서의 조건
1. **명확성**: 누구나 읽고 이해할 수 있어야 함
2. **완결성**: 필요한 정보가 모두 포함되어 있어야 함
3. **일관성**: 전사 스타일 가이드를 따라야 함
4. **최신성**: 정기적으로 업데이트되어야 함
5. **검색 가능성**: 적절한 키워드와 링크가 포함되어야 함

### 피해야 할 것
- ❌ 구두 설명만으로 끝내기 (반드시 문서화)
- ❌ 개인 로컬에만 보관 (공유 폴더에 저장)
- ❌ 버전 표기 없이 덮어쓰기 (변경 이력 기록)
- ❌ 링크 없이 "○○ 문서 참고" 표기
- ❌ 메타 정보 누락
