## 업무 지시 후 흐름

모든 에이전트는 아래 흐름을 따릅니다 (COMMON_RULES.md):

```
나: 업무 지시
     ↓
에이전트: 📌 이해 보고서 출력 (해석 + 범위 + 산출물 + 확인 질문)
     ↓
나: "확인" 또는 "수정해줘"
     ↓
에이전트: task.md 생성 → 대표에게 보고
     ↓
나: task.md 확인 후 "진행해" 또는 "수정해줘"
     ↓
에이전트: task.md 체크리스트 기반 실행
     ↓
에이전트: 📋 완료 보고 (원본 지시 + 결과 + 위임 사항)
```

**"바로 진행해"** 라고 말하면 이해 보고서를 건너뛰고 task.md부터 시작합니다.
단, task.md 승인 단계는 생략할 수 없습니다.

## 자주 쓰는 지시 예시

### 새 기능 개발
```
[자비스 세션]
"사용자 로그인 기능 만들어줘. 구글 OAuth 사용."
→ 이해 보고서 → 승인 → PRD → 설계 → 개발 → QA 요청
```

### 코드 리뷰
```
[김감사 세션]
"src/main.js 코드 리뷰해줘."
→ 이해 보고서 → 승인 → 기능/보안/UX 병렬 QA → 통합 보고서
```

### 리팩토링
```
[강철 세션]
"src/api/ 폴더 코드 중복 제거해줘."
→ 이해 보고서 → 승인 → Before/After 분석 → 리팩토링 실행
```

### 기획/분석
```
[벙커 세션]
"경쟁사 3곳 분석해서 우리 제품 포지셔닝 정리해줘."
→ 이해 보고서 → 승인 → 데이터 분석 → 보고서 작성
```

### 문서 정리
```
[꼼꼼이 세션]
"API 문서를 템플릿에 맞게 정리해줘."
→ 이해 보고서 → 승인 → 스타일 가이드 기반 정리 → 배포
```

## 동시 멀티 세션 운영

여러 창을 열고 동시에 작업할 때:

```
창 1: [자비스] "로그인 기능 개발해줘"     → 개발 진행 중
창 2: [김감사] "대시보드 코드 리뷰해줘"    → QA 진행 중
창 3: [꼼꼼이] "API 문서 템플릿 만들어줘"  → 문서 작업 중
```

각 창으로 돌아오면 **📋 완료 보고**에 원본 지시가 포함되어 있어
어떤 지시를 했는지 바로 파악 가능합니다.

## 역할 경계 규칙

에이전트는 자기 역할 밖의 일을 하지 않습니다.

```
김감사가 버그를 발견하면:
  ❌ 직접 코드 수정 (금지)
  ✅ QA 보고서 작성 + "자비스 팀에 수정 요청" 위임 안내

강철이 리팩토링 중 새 기능이 필요하면:
  ❌ 직접 기능 개발 (금지)
  ✅ 리팩토링 완료 + "자비스 팀에 기능 추가 요청" 위임 안내
```
