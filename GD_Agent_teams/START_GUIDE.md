# GD Agent Teams — 시작 가이드

> **이 문서 하나만 읽으면** 에이전트 팀을 바로 사용할 수 있습니다.

---

## 이게 뭔가요?

AI 에이전트 5개 팀(총 21명)이 각자 역할에 맞게 일하는 시스템입니다.
코드 개발, QA, 리팩토링, 기획, 문서 작업을 팀별로 분담합니다.

**핵심 규칙 3가지:**
1. 에이전트는 **자기 역할 밖의 일을 하지 않습니다** (위임만 합니다)
2. 모든 업무는 **"이해 보고 → 승인 → 실행"** 순서로 진행됩니다
3. 완료 시 **누가 뭘 했는지 + 토큰 사용량**까지 보고합니다

---

## 3단계로 시작하기

### STEP 1. 폴더 준비

```
프로젝트 루트에 GD_Agent_teams/ 폴더가 있는지 확인합니다.
없으면 다른 프로젝트에서 복사해 옵니다.
```

### STEP 2. Claude Code 세션 열기

새 터미널에서 Claude Code 세션을 엽니다.
**팀마다 별도 세션**을 사용하세요.

### STEP 3. 아래 프롬프트 복사 → 붙여넣기

사용할 팀의 프롬프트를 **그대로 복사**해서 입력합니다. 끝.

---

## 어떤 팀을 부를까?

```
"코드를 만들거나 고쳐야 해"        → 🤵 자비스 개발팀
"코드 품질을 검사하고 싶어"        → 🕵️ 김감사 QA팀
"코드를 정리하거나 성능을 올리고 싶어" → 🔧 강철 AX팀
"기획이나 분석이 필요해"           → 🏴 벙커 팀
"문서를 정리하거나 템플릿이 필요해"  → 📝 꼼꼼이 문서팀
```

잘 모르겠으면? → 아무 팀이나 불러서 물어보세요.
자기 역할이 아니면 알아서 담당 팀을 안내해줍니다.

## 업무 진행 흐름 (5단계)

에이전트를 호출한 뒤, 업무를 지시하면 아래 흐름으로 진행됩니다.

```
┌─────────────────────────────────────────────────┐
│                                                 │
│  나: 업무 지시                                    │
│       ↓                                         │
│  에이전트: 📌 이해 보고서                          │
│       ↓                                         │
│  ★ 나: "진행해" ← 1차 승인                        │
│       ↓                                         │
│  에이전트: 📝 task.md 생성 → 나에게 보여줌           │
│       ↓                                         │
│  ★ 나: "진행해" ← 2차 승인                        │
│       ↓                                         │
│  에이전트: 실행 (task.md 체크리스트 기반)             │
│       ↓                                         │
│  에이전트: 📋 완료 보고                             │
│       ↓                                         │
│  (업로드 필요 시) 에이전트: 🚀 업로드 제안            │
│       ↓                                         │
│  ★ 나: "업로드해" ← 3차 승인                       │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 내가 해야 할 일 (★ 표시)

| 시점 | 내가 할 일 | 말하면 되는 것 |
|------|----------|-------------|
| 이해 보고서 확인 후 | 1차 승인 | "진행해", "확인", "ㅇㅇ" |
| task.md 확인 후 | 2차 승인 | "진행해", "확인", "ㅇㅇ" |
| 업로드 제안 확인 후 | 3차 승인 | "업로드해", "push해", "ㅇㅇ" |
| 수정이 필요하면 | 수정 요청 | "이 부분 수정해줘: ..." |
| 이해 보고 건너뛰고 싶으면 | 빠른 시작 | **"바로 진행해"** (task.md 승인은 필수) |

## 완료 보고서에서 확인할 것

업무가 끝나면 아래 형식의 보고서가 나옵니다:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 업무 완료 보고
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【원본 지시】    ← 내가 뭘 시켰는지 원문 그대로
【에이전트 이해 요약】 ← 핵심 요청 + 범위
【수행 결과】    ← 완료된 항목 체크리스트
【산출물】      ← 생성/수정된 파일 경로
【위임 사항】    ← 다른 팀에 넘겨야 할 일 (없으면 "해당 없음")
【팀원별 수행 내역】 ← 누가 뭘 했는지 (미참여 팀원도 표시)
【토큰 사용량】   ← 입력/출력/총 토큰 + 세션 시간
【task.md 상태】  ← 파일 경로 + ✅ 완료
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**확인 포인트:**
- 【위임 사항】이 있으면 → 해당 팀 세션을 열어서 위임된 업무를 지시
- 【토큰 사용량】으로 비용 추적
- 【팀원별 수행 내역】으로 업무 분배 확인

## 멀티 세션 운영

여러 팀을 동시에 돌릴 수 있습니다.

```
터미널 1: [자비스] "로그인 기능 개발해줘"      → 개발 중...
터미널 2: [김감사] "대시보드 코드 리뷰해줘"     → QA 중...
터미널 3: [꼼꼼이] "API 문서 템플릿 만들어줘"   → 문서 작업 중...
```

각 창에 돌아오면 **완료 보고서에 원본 지시가 포함**되어 있어
어떤 작업을 시켰는지 바로 파악 가능합니다.

**업무 연계 예시:**

```
1. [자비스] 로그인 기능 개발 완료
       ↓ 위임: "김감사 팀에 코드 리뷰 요청"
2. [김감사] 로그인 코드 리뷰 → QA 보고서 작성
       ↓ 위임: "강철 팀에 리팩토링 요청"
3. [강철] 로그인 코드 리팩토링
       ↓ 위임: "꼼꼼이 팀에 API 문서 업데이트 요청"
4. [꼼꼼이] API 문서 정리
```

## 역할 경계 — 이것만 기억하세요

에이전트는 **자기 역할 밖의 일을 절대 하지 않습니다.**
대신 담당 팀을 안내해줍니다.

```
김감사가 버그를 발견하면:
  ❌ "제가 고쳤습니다" (금지)
  ✅ "자비스 팀에 수정 요청 바랍니다" (위임)

강철이 리팩토링 중 새 기능이 필요하면:
  ❌ "제가 기능도 만들었습니다" (금지)
  ✅ "자비스 팀에 기능 추가 요청 바랍니다" (위임)
```

이렇게 해야 **누가 뭘 했는지 추적**이 가능합니다.

## 파일이 수정되면

에이전트가 파일을 수정하면:

1. **파일 상단에 배포 헤더**가 자동 추가/업데이트됩니다
   - 버전, 날짜, 수정한 에이전트, 변경 내용 등
2. **업로드가 필요하면** 자동으로 하지 않고 먼저 물어봅니다
   - "🚀 로컬 저장 완료했습니다. 업로드할까요?"
3. 승인하면 그때 업로드합니다 (clasp push, git push 등)

## 트러블슈팅

### 에이전트가 규칙을 안 따를 때

```
"COMMON_RULES.md의 task_protocol을 따라줘. 이해 보고서부터 작성해."
```

### 에이전트가 역할 밖의 일을 할 때

```
"역할 경계를 지켜줘. 코드 수정은 자비스 팀 소관이야. 위임 안내만 해줘."
```

### 에이전트가 task.md 없이 바로 실행할 때

```
"task.md를 먼저 보여주고 내 승인을 받아줘."
```

### 에이전트가 승인 없이 push/업로드 할 때

```
"업로드하기 전에 나한테 먼저 제안하고 승인받아줘."
```

### 완료 보고서에 토큰/팀원 내역이 없을 때

```
"완료 보고서에 【팀원별 수행 내역】이랑 【토큰 사용량】도 포함해줘."
```

## 한 줄 요약

```
1. 팀 호출 프롬프트 복사 → 붙여넣기
2. 업무 지시
3. "진행해" 2번 말하기 (이해 보고서 승인 + task.md 승인)
4. 완료 보고서 확인
5. 필요하면 업로드 승인
```

이게 전부입니다.
