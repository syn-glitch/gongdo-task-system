# 16단계 Implementation Plan: 프로젝트 및 업무 분류 체계 고도화 (Advanced Project & Task Labeling)

> **최종 작성일**: 2026-02-24  
> **목표**: AI가 업무의 맥락을 정확히 이해하고 자동화하기 위한 정밀한 인식 라벨링 및 구조적 분류 시스템 구축

---

## 1. 도입 배경 및 설계 철학

### ❌ 현재 문제점: 비정형 데이터의 한계
- 프로젝트명이 자유 입력으로 되어 있어 데이터 정합성(오타 등)이 낮음.
- 업무 ID가 단순 숫자로 되어 있어 ID 자체만으로는 업무의 맥락(Context)을 파악하기 어려움.
- AI가 '업무량'이나 '워크플로우'를 분석할 때 참고할 프로젝트별 상세 배경 지식이 부재함.

### ✅ 개선 방향: AI 친화적 구조화 (AI-Friendly Structuring)
- **정형화된 라벨링**: 프로젝트 코드를 ID의 일부로 편입하여 데이터의 자기 설명성(Self-describing) 강화.
- **컨텍스트 엔지니어링**: AI가 프로젝트의 목적을 학습할 수 있는 '설명(Description)' 메타데이터 도입.
- **데이터 통제**: 입력 단계에서부터 프로젝트를 선택형으로 제한하여 데이터 오염 방지.

---

## 2. 상세 구현 프로세스 (4단계)

### Step 1. `Projects` 시트 속성 확장 및 AI 학습 데이터 추가
- **관리 항목 추가**:
  - `Project Description (AI용)`: 해당 프로젝트의 목표, 주요 마일스톤, 연관 인물 등 (AI가 리포트 생성 시 참고).
  - `Project Status`: 활성/대기/완료 (AI가 현재 진행 중인 업무만 필터링하는 기준).
  - `Priority`: 중요도 (업무량 분석 및 대시보드 시각화 시 가중치 부여).

### Step 2. 구조적 업무 ID (Structured Task ID) 시스템 도입
- **변환 로직**: 기존 `ID: 1` -> `ID: GONG-001` 형태로 업데이트.
- **자동화**: 새로운 업무 생성 시 사용자가 선택한 프로젝트의 코드를 바탕으로 일련번호를 조합하는 GAS 로직 구현.
- **기존 데이터 마이그레이션**: 이전 데이터들을 새로운 ID 체계로 일괄 치환하는 스크립트 실행.

### Step 3. 입력 UX 고도화 (Selection over Collection)
- **슬랙 모달**: '프로젝트명' 입력 필드를 `static_select`(드롭다운)로 변경. `Projects` 시트의 상위 목록을 실시간으로 로드.
- **웹 앱**: Judy Note 에디터 및 업무 등록 창에서도 동일한 드롭다운 시스템 적용.
- **효과**: "공도 개발", "공도 프로젝트" 등으로 파편화되던 데이터를 하나로 통합.

### Step 4. AI 컨텍스트 리포팅 시스템 구축 (Intelligence Analysis)
- **로직**: AI 리포트 생성 시 `Projects` 시트의 설명을 프롬프트에 주입.
- **활용**:
  - "이번 달 GONG 프로젝트의 업무 부하가 높습니다. 신규 인력 배치가 필요해 보입니다."
  - "MKT 프로젝트는 기한 대비 진척도가 20%로 낮습니다. 확인이 필요합니다."

---

## 3. 기술적 기대 효과

- **대시보드 정확도**: 프로젝트 코드를 기준으로 자동 집계가 가능해져 시각적 요약의 신뢰도 상승.
- **자동화 워크플로우**: 프로젝트 코드별로 다른 자동화 규칙(예: MKT 프로젝트는 완료 시 특정 채널에 자동 공지) 적용 가능.
- **RAG 효율성**: AI가 과거 기록 검색 시 프로젝트 코드를 키워드로 활용하여 검색 정확도 대폭 향상.

---

## 4. 검증 시나리오 (Verification Plan)

1. 슬랙 모달에서 신규 프로젝트를 선택하고 등록했을 때, ID가 `[코드]-[번호]` 형식으로 자동 생성되는가?
2. `Projects` 시트에 새로운 프로젝트를 추가하면 슬랙/웹 드롭다운 메뉴에 자동으로 반영되는가?
3. AI에게 특정 프로젝트의 상태를 물었을 때, `Projects` 시트에 입력된 상세 설명을 기반으로 답변하는가?
4. 고정된 프로젝트 코드를 기반으로 대시보드의 차트와 그래프가 정상적으로 업데이트되는가?
