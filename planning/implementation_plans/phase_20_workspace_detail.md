# 🚀 Judy Workspace (통합 앱) 구축 계획서

## 1. 개요 및 배경
**팀원 피드백:** "주디 노트(메모/기록)와 대시보드(업무 관리)가 왜 분리되어 있는지 모르겠다. 노트를 적고 다시 업무로 등록해야 해서 중복 작업의 느낌이 든다."
**해결 방향:** 두 개의 분리된 웹앱을 완전히 하나의 **Single Page Application (SPA)** 로 통합한 **"Judy Workspace"**를 구축하여, 생각의 기록부터 업무 등록 및 추적까지 "끊김 없는 흐름(Seamless Flow)"을 제공합니다.

---

## 2. 통합 아키텍처 및 UI/UX 설계

### 2.1 화면 구성 (SPA 구조)
새로고침 없이 탭으로 즉각 전환되는 부드러운 UI를 채택합니다.
- **상위 글로벌 네비게이션 (GNB)**
  - 로고 및 유저 환영 문구
  - **[📝 노트 및 회의록]** / **[📊 업무 대시보드]** 탭 (클릭 시 화면 즉시 전환)
  - ☀️/🌙 다크 테마 전환 버튼 (전역 공유)
- **노트 탭 (기존 judy_note.html 영역)**
  - 좌측: 과거 폴더/파일 트리 메뉴
  - 우측: 광활한 마크다운 에디터 
  - ⭐ **강화 포인트:** 노트에서 `✨ AI 업무 추출`을 누른 직후, 추출된 결과가 백그라운드에서 즉시 대시보드로 넘어가 반영되도록 연결. 
- **대시보드 탭 (기존 task_dashboard.html 영역)**
  - 상단: 진행/완료/대기 요약 집계 카드
  - 중앙: 필터링된 업무 리스트 및 상태 드롭다운
  - 상세: 업무 행 클릭 시 나타나는 팝업 모달

### 2.2 인증 및 라우팅 통합
- **단일 매직 링크:** 슬랙에서 받는 하나의 URL 토큰으로 접속하면 노트와 대시보드 권한을 동시에 획득합니다.
- **단일 백엔드 통신:** `google.script.run`을 통한 분산된 API 호출 관리가 하나로 통합되어 유지보수성이 극대화됩니다.

---

## 3. 기대 효과 (팀원들의 니즈 충족)
1. **맥락 유지 (Context Switching Zero):** 회의록을 적다가 즉시 탭을 넘겨 내 할 일 목록을 점검할 수 있습니다.
2. **"중복 작업" 느낌 해소:** 일기장(노트)에 텍스트로 적고 AI 추출을 누른 뒤, 바로 옆 대시보드 탭으로 넘어가면 할 일이 이미 예쁘게 셋업되어 있는 마법(Magic)을 경험합니다.
3. **학습 비용 최소화:** "메모는 저기서 링크 누르고, 업무는 여기서 치고" 같은 혼란이 사라집니다. 북마크 하나만 해두면 모든 업무가 해결됩니다.

---

## 4. 개발 및 마이그레이션 전략
- **안정성:** 기존 코드(`judy_note`, `task_dashboard`)가 이미 모듈화가 잘 되어 있으므로, 새로운 껍데기(`judy_workspace.html`)를 만들고 안에 컴포넌트 형태로 끼워 넣는 방식으로 빠르고 안전하게 개발합니다. 기존 백엔드(`web_app.gs`)의 함수는 그대로 재사용합니다.
- **Roll-out:** 통합 앱 개발이 끝나고 충분한 사내 테스트를 거친 후, 하루 날을 잡고 슬랙 연결 링크를 새 앱으로 스위칭하여 배포합니다.
