## 1. 프로젝트 개요 (Overview)
기존의 복잡한 요구사항 정의서나 구글 시트를 건너뛰고, 비전공자 팀원이 **"WOW 매니저"와 카카오톡처럼 대화**하는 것만으로 실행 가능한 태스크(Task Package)를 뽑아내는 **프론트엔드 채팅 관문(Gateway)**을 구축합니다.

### 🎯 핵심 목표
- **접근성 극대화**: 누구나 쉽게 아이디어를 제안할 수 있는 대화형 UI (`wow_chat.html`)
- **파이프라인 자동화**: [채팅 인터뷰] → [기획안 생성] → [하네스/벙커 태스크 전환]의 심리스 연결

## 2. 세부 태스크 및 담당자 배분

### Phase 1: WOW 전용 챗봇 UI 프론트엔드 개발
비전공자가 가장 먼저 마주할 직관적인 채팅 화면을 만듭니다.

- [ ] **[Task 1.1] 챗봇 UI 레이아웃 및 디자인 (모바일 반응형)**
  - **담당자**: 🎨 벨라 (UX/UI 디자이너), 👧 클로이 (Frontend Dev)
  - **내용**: 
    - 메시지 버블(말풍선) UI, 입력창, 전송 버튼
    - 테마(Dark/Light) 스위치 적용
    - 현재 대화 상태(예: "공감 중", "구체화 중") 인디케이터 상단 표시
    - **(QA 반영)** 입력값 검증 로직 (최대 500자 제한, XSS 방지 특수문자 필터링)
- [ ] **[Task 1.2] 에이전트 페르소나 적용 및 마이크로 인터랙션**
  - **담당자**: 👧 클로이 (Frontend Dev)
  - **내용**: WOW 매니저 프로필 이미지 적용, 상대방이 타이핑 중일 때 `...` 애니메이션 표출

### Phase 2: 대화 엔진 및 데이터 파이프라인 연동
채팅에서 오고 간 대화를 상태 데이터로 만들어 하네스(V4 시스템)로 넘깁니다.

- [ ] **[Task 2.1] 채팅 세션 상태 관리 (Session Management)**
  - **담당자**: 👨‍💻 알렉스 (Tech Lead), 👩‍💻 에이다 (Backend Dev)
  - **내용**: 
    - 대화 내역(Context)을 임시 저장하는 `wow_session.json` 구조 설계
    - 5단계 인터뷰 프레임워크에 따른 단계 전환 로직
    - **(QA 반영)** 무한 대화 방지: 채팅 턴(Turn) 수 최대 7회 제한, 초과 시 오프라인 미팅 에스컬레이션
    - **(QA 반영)** 좀비 세션 파기: 1시간 이상 미활동 세션 자동 만료 (Garbage Collection) 또는 SessionStorage 초기화
- [ ] **[Task 2.2] 완료 핸드오프(Handoff) 및 Webhook 연동**
  - **담당자**: 👩‍💻 에이다 (Backend Dev)
  - **내용**: 
    - 대화가 [확인] 단계에서 종료되면, **쉬운설명서**와 **연결이**의 산출물 포맷으로 변환
    - 변환된 데이터를 V4 `agent_sync.gs`의 Webhook으로 쏴서 시트에 자동 등록(`PLANNING` 상태)
    - **(QA 반영)** Webhook 통신 실패 대비 Fallback UI: 실패 시 에러 메시지 표출 및 '재전송(Retry)' 버튼 제공
