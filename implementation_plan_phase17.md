# 17단계 Implementation Plan: 역할 기반 업무 뷰 및 팀원 협업 워크플로우 (Role-Based Views & Team Collaboration)

> **최종 작성일**: 2026-02-24  
> **선행 조건**: 16단계(프로젝트 분류 체계 고도화)의 구조적 ID, 드롭다운 입력 통제가 완료된 상태  
> **목표**: 정형화된 데이터를 기반으로, 대표/팀장/팀원 각각에게 최적화된 업무 뷰와 협업 워크플로우를 제공

---

## 1. 도입 배경

### ❌ 현재 문제점
- `Tasks` 시트에 모든 업무가 쌓이지만, 팀원 개인이 "내 업무만" 빠르게 확인할 수 있는 채널이 없음.
- 대표/팀장이 팀원별 업무 현황을 파악하려면 시트를 직접 열어 수동 필터링해야 함.
- 팀원 간 업무 요청/위임 시 구두나 슬랙 일반 메시지에 의존하여 추적이 불가능함.

### ✅ 개선 방향
- **"데이터는 하나, 보는 눈(View)은 역할마다 다르게"** 원칙 적용.
- `Users` 시트에 역할(Role) 및 소속 팀(Team) 속성을 추가하여, AI가 권한과 맥락에 맞는 응답을 자동 생성.

---

## 2. 상세 구현 프로세스 (4단계)

### Step 1. `Users` 시트 확장 및 역할 체계 도입
- **추가 컬럼**:
  - `역할 (Role)`: 대표 / 팀장 / 팀원 → 리포트 접근 범위 및 알림 수준 차등 적용.
  - `소속 팀 (Team)`: 개발 / 마케팅 / 운영 등 → 팀별 업무량 분석 기준.
  - `기본 프로젝트 (Default Project)`: 슬랙 모달에서 프로젝트 기본값으로 자동 세팅.
- **`Config` 시트 신설**: 시스템 전역 설정(관리자 목록, 알림 채널 ID 등) 관리.

### Step 2. 역할 기반 업무 뷰 시스템 구현 (Role-Based View)
- **팀원 개인 뷰**: `/주디 내 업무` → AI가 `담당자 = 본인`으로 필터링 후 상태별 리스트 반환.
- **팀장 관리 뷰**: `/주디 팀 현황` → AI가 `소속 팀 = 팀장의 팀`에 해당하는 전체 업무를 담당자별로 그룹핑하여 요약.
- **대표 전사 뷰**: `/주디 전사 현황` → 프로젝트별 진척률, 팀별 업무 부하, 지연 업무 하이라이트를 포함한 경영 대시보드 제공.

### Step 3. 팀원 간 업무 요청/위임 워크플로우 구축
- **업무 요청 플로우**:
  1. `/주디 요청 @이지은 홈페이지 배너 디자인` 입력.
  2. `Tasks` 시트에 `담당자=이지은`, `요청자=본인`으로 자동 등록.
  3. 이지은에게 슬랙 DM으로 "새로운 업무 요청이 도착했습니다" 알림 발송.
- **완료 알림 플로우**:
  1. 이지은이 해당 업무의 상태를 `완료`로 변경.
  2. 요청자(본인)에게 "요청하신 업무가 완료되었습니다" 자동 DM 전송.
- **업무 위임**: 담당자 변경 시 기존/신규 담당자 모두에게 알림.

### Step 4. 모닝 브리핑 개인화 및 팀별 자동 리포트
- **기존 모닝 브리핑(6단계) 확장**: 역할에 따라 브리핑 내용을 차등 제공.
  - 팀원: "오늘 마감인 업무 N건, 진행 중 N건"
  - 팀장: "우리 팀 오늘 마감 N건, [홍길동] 업무 지연 1건 주의"
  - 대표: "전사 프로젝트 진척률 요약 + 이번 주 완료 예정 주요 마일스톤"

## 3. 팀 협업 시나리오 예시

### 시나리오: 마케팅 캠페인 프로젝트 (MKT)

```
[팀장: 정혜림]
  /주디 MKT 현황
  → "MKT 프로젝트: 진행 중 3건, 대기 2건
      - 이지은: 홍보 포스터 제작 (진행중, 마감 2/25)
      - 문유나: 요청사항 파악 (대기)
      - 이상호: 교사 DB 확인 (대기)"

[팀원: 이지은]
  /주디 내 업무
  → "진행 중: 홍보 포스터 제작 (MKT-003, 마감 2/25)
      대기: 넷마블 강사 모집 홍보 (MKT-002)"

[팀원 간 요청]
  송용남: /주디 요청 @이지은 배너 사이즈 1200x630으로 수정
  → 이지은 DM: "📩 송용남님이 새 업무를 요청했습니다: 배너 사이즈 수정"
```

## 4. 검증 시나리오 (Verification Plan)

1. 각 팀원이 `/주디 내 업무`를 입력했을 때, 본인의 업무만 정확히 필터링되어 표시되는가?
2. 팀장이 `/주디 팀 현황`을 입력했을 때, 자신의 팀원들의 업무만 묶여서 요약되는가?
3. `/주디 요청 @팀원 업무내용` 시 `Tasks` 시트에 요청자/담당자가 정확히 기록되고, 해당 팀원에게 DM 알림이 가는가?
4. 업무 상태 변경(완료) 시 요청자에게 자동 완료 알림이 발송되는가?
5. 모닝 브리핑이 역할(대표/팀장/팀원)에 따라 다른 내용으로 전송되는가?
