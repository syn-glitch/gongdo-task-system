## 원인 추정

### 문제 1: 상단 메뉴 중복
- **파일**: `Code.gs` 내 `showHomePage()` 또는 네비게이션 렌더링 관련 함수
- **함수**: 추정 `renderNavigation()` 또는 HTML 템플릿 include 로직
- **원인**: 
  - GAS HTML Service에서 `<?!= include() ?>` 구문이 중복 호출되고 있을 가능성
  - 조건부 렌더링 로직(`if/else`) 누락으로 같은 메뉴가 두 번 삽입
  - 템플릿 파일(`Navigation.html`)이 두 곳에서 중복 include되는 구조적 오류

### 문제 2: 캘린더 텍스트 겹침
- **파일**: `Calendar.html` 및 `Calendar.gs` (캘린더 데이터 fetch 함수)
- **함수**: 추정 `getEventsForDate(date)`, 날짜 셀 렌더링 로직
- **원인**:
  - 하루에 3개 이상의 이벤트가 적재될 경우 CSS `height` 제한 없이 모두 표시 시도
  - `z-index` 미설정으로 텍스트 레이어가 클릭 영역을 덮음
  - `pointer-events: none` 속성이 잘못된 요소에 적용되어 클릭 차단
  - 1280x800 해상도에서 캘린더 그리드 너비 계산 오류 (반응형 미대응)

## 영향 범위

**직접 영향**
- v3.1.0 사용자 전체에게 홈 화면 메뉴 중복 노출
- 캘린더에서 이벤트가 많은 날짜(3개 이상)의 클릭 불가
- 1280x800 이하 해상도 사용자(약 30% 추정)의 캘린더 사용성 저하

**간접 영향**
- 신규 사용자 첫 인상 악화 → 이탈률 증가 우려
- 캘린더를 통한 일정 생성/수정 워크플로우 차단 → 다른 경로(목록 뷰) 사용 강제
- 중복 메뉴로 인한 DOM 크기 증가 → 페이지 로딩 속도 미세 저하

## 수정 계획

### 1단계: 긴급 핫픽스 (메뉴 중복 제거)
1. **`Code.gs` 파일 점검**
   - `showHomePage()` 함수 내 `<?!= include('Navigation') ?>` 호출 횟수 확인
   - 중복 include 구문 제거 또는 조건부 렌더링 추가
   
2. **HTML 템플릿 구조 검증**
   - `Home.html`, `Header.html` 등에서 Navigation 컴포넌트 중복 삽입 여부 확인
   - 하나의 템플릿에서만 include되도록 리팩토링

### 2단계: 캘린더 수정
1. **이벤트 표시 개수 제한**
   - `Calendar.gs`의 날짜별 이벤트 조회 함수에 `.slice(0, 3)` 적용
   - 3개 초과 시 "+N개 더보기" 텍스트로 대체 (클릭 시 모달 팝업)

2. **CSS 개선** (`Calendar.html` 또는 스타일시트)
   ```css
   .calendar-day-cell {
     overflow: hidden;
     position: relative;
     z-index: 1;
   }
   .calendar-event-item {
     text-overflow: ellipsis;
     white-space: nowrap;
     pointer-events: auto;
     z-index: 2;
   }
   ```

3. **반응형 대응**
   - 1280x800 해상도에서 캘린더 그리드 너비 재계산 로직 추가
   - 미디어 쿼리로 작은 화면에서 폰트 크기/패딩 조정

### 3단계: 회귀 테스트
- 1920x1080, 1366x768, 1280x800 해상도에서 캘린더 레이아웃 검증
- Chrome, Safari, Firefox 브라우저 테스트
- 이벤트 0개/1개/5개/10개 날짜에서 클릭 동작 확인
