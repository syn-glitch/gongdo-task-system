**회신 기한**: 2026-02-29 (금) 18:00까지

## 1. 개요

공도 업무 관리 시스템의 확장과 고도화에 따라, 각 에이전트 팀의 전문성을 명확히 하고 대외적인(팀장님 및 외부 감사) 신뢰도를 높이기 위해 **팀 소개 및 스킬셋 표기 방식을 표준화**하고자 합니다.

## 2. 주요 변경 및 요청 사항

### A. 팀원 소개 포맷 통일

자비스 팀의 [AI_AGENT_TEAM_OVERVIEW.md](../../docs/architecture/AI_AGENT_TEAM_OVERVIEW.md) 스타일에 맞춰, 김감사 QA 팀의 문서도 아래의 항목을 포함하도록 업데이트를 완료하였습니다. 앞으로 인원 충원이나 역할 변경 시 이 포맷을 유지해 주시기 바랍니다.

- **한줄 소개**: 에이전트의 정체성을 나타내는 페르소나 정의
- **핵심 역량 및 스택**: 보유 기술 및 담당 도메인
- **담당 업무 (R&R)**: 구체적인 책임 범위
- **장착된 전문 스킬 (Skills)**: 에이전트가 사용하는 실제 기술의 **공식 문서(URL)** 명시
- **협업 방식**: 타 에이전트와의 상호작용 방식

### B. 전문 스킬(Skills) 출처 명시 (중요)

에이전트의 환각(Hallucination)을 방지하고 기술적 근거를 확보하기 위해, 모든 스킬에는 반드시 **공식 Reference URL**을 첨부해야 합니다.

- 예: `OWASP Top 10`, `WCAG 2.1`, `ISTQB` 등 공식 표준 사이트 링크
- **검증 프로세스**: 신규 스킬 추가 시 알렉스 (Tech Lead)에게 사전 검증 요청
- **유효성 점검**: 연 1회 (매년 1월) 전체 스킬 링크 유효성 점검 실시

### C. 협업 팀(External Teams) 섹션 추가

팀 간의 유기적인 연결을 위해 본인의 팀 문서에서 상대 팀 문서를 상호 참조하는 섹션을 추가하였습니다.

### D. 문서 변경 이력(Changelog) 추가

향후 문서 변경 추적을 위해 각 팀 문서 하단에 변경 이력 테이블을 추가합니다.

```markdown
## 📝 문서 변경 이력
| 버전 | 날짜 | 변경자 | 주요 변경 사항 |
|------|------|--------|----------------|
| v2.1 | 2026-02-27 | 자비스 팀 | 스킬 출처 URL 추가, 협업 팀 섹션 신설 |
```

## 5. 협업 인터페이스 정의 (제안)

양 팀 간 명확한 산출물 교환을 위해 아래와 같은 표준을 제안합니다.

### 자비스 팀 → 김감사 QA 팀

- **인입 산출물**: `qa/qa_requests/{날짜}_feature_name.md` (QA 요청서)
- **포함 정보**: GitHub PR 링크, 변경 파일 목록, 테스트 시나리오
- **전달 방식**: QA 요청 슬랙 채널 알림 + 문서 링크

### 김감사 QA 팀 → 자비스 팀

- **산출물**: `qa/qa_reviews/{날짜}_integrated_report.md` (통합 QA 보고서)
- **포함 정보**: 승인/반려 결정, Critical/High/Medium 이슈 목록, 수정 권고사항
- **전달 방식**: 슬랙 멘션 + GitHub PR 코멘트

### 에스컬레이션 조건 (팀장님 승인 필요)

다음의 경우 양 팀장이 협의 후 팀장님(USER)께 승인 요청:

- 두 팀 간 표준 충돌 발생 시
- 새로운 스킬셋 카테고리 추가 시 (예: 성능 테스트 전문가 영입)
- 문서 구조 대폭 변경 시 (예: 새로운 워크플로우 섹션 추가)

## 6. 기대 효과 및 성공 지표

### 정성적 효과

- **전문성 강화**: 공식 출처 기반 스킬로 대외 신뢰도 향상
- **협업 효율화**: 명확한 인터페이스로 커뮤니케이션 오버헤드 감소
- **지속 가능성**: 일관된 문서 포맷으로 신규 팀원 온보딩 시간 단축

### 측정 가능한 지표 (3개월 후 측정)

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 환각 방지율 | 스킬 출처 명시로 잘못된 API 사용 0건 | 알렉스 코드 리뷰 로그 |

| 문서 일관성 | 양 팀 문서 포맷 100% 통일 | 분기별 문서 감사 |
