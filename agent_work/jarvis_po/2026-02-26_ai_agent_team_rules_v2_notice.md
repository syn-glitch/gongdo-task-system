## 🎯 v2.0 주요 변경사항 (Top 5)

### 1️⃣ 작업 분류 체계 도입 ⭐ NEW
작업을 3단계로 분류하여 차등 프로세스 적용:

| 분류 | 소요 시간 | 프로세스 | 예시 |
|:---|:---:|:---|:---|
| 🟢 Micro | < 30분 | Fast-Track (간소화) | 오타 수정, CSS 조정 |
| 🟡 Small | 30분~2시간 | Simplified (부분 간소화) | 버그 수정, 단일 기능 추가 |
| 🔴 Medium+ | 2시간+ | Full Process (전체) | Phase 기능, 아키텍처 변경 |

**영향**:
- Micro 작업은 사전 계획서 생략 가능
- Fast-Track으로 즉시 구현 후 사후 보고

---

### 2️⃣ 회고 규칙 대폭 개선 ⭐ 70% 절감

**v1.0**: 모든 작업마다 개별 회고 문서 작성 (30분~1시간)
**v2.0**: 개별 회고 폐지 → **정기 회고 체계 (일간/주간/월간)**

#### 📅 일간 회고 (5분)
- 매일 EOD에 채팅창에 3줄 보고
- 파일로 저장하지 않음
```
[Ada] 오늘 회고 (2026-02-27)
- ✅ 완료: 주디노트 즐겨찾기 백엔드 (5시간)
- 📚 배운 점: LockService 타임아웃은 10초 이상 필요
- 🚀 내일 할 일: 알림 시스템 백엔드 구현 계획
```

#### 📊 주간 회고 (30분)
- 매주 금요일 오후
- 전체 팀 참여 (자비스, Ada, Chloe, 김감사, 벨라, 팀장님)
- 파일: `qa/retrospectives/weekly/YYYY-WW_weekly_retro.md`
- Keep/Problem/Try 패턴

#### 🗓️ 월간 회고 (1시간)
- 매월 마지막 금요일
- 전체 팀 필수 참여
- 파일: `qa/retrospectives/monthly/YYYY-MM_monthly_retro.md`
- 아키텍처 결정 회고 (ADR), 성장 메트릭, 다음 달 로드맵

**예외**: 특별히 중요한 Phase는 선택적으로 개별 회고 가능

---

### 3️⃣ 에러 메시지 이원화 ⭐ 자비스 제안 수용

**사용자 노출**: 한글
```javascript
showToast('즐겨찾기 설정에 실패했습니다. 잠시 후 다시 시도해주세요.');
```

**시스템 내부**: 영문 식별자 + 한글 설명
```javascript
throw new Error('ERR_LOCK_TIMEOUT: 동시 작업 대기 시간 초과');
```

**장점**:
- 글로벌 검색 가능
- 비전공자도 이해 가능

---

### 4️⃣ 진행률 공유 최적화 ⭐ 자비스 제안 수용

**PROGRESS.md 업데이트 빈도**:
- ❌ Before: 실시간 (10회/일)
- ✅ After: 필요 시 (2회/일)
  - 작업 시작 시
  - 마일스톤 완료 시
  - 차단 상황 발생 시
  - EOD (End of Day)

**에디터 출력 최적화**:
- ❌ Before: 전체 코드 파일 출력
- ✅ After: Diff + 링크만
  - 토큰 사용량 **90% 절감**

---

### 5️⃣ Fast-Track 프로세스 도입 ⭐ 자비스 제안 수용

**적용 대상**: 🟢 Micro 작업 (< 30분)

**프로세스**:
```
1. 채팅창에 즉시 보고:
   "[Ada] Fast-Track: USER_GUIDE.md 오타 수정 진행합니다."

2. 구현 완료 후 보고:
   "[Ada] Fast-Track 완료: USER_GUIDE.md 오타 수정
   - 소요 시간: 5분
   - 커밋: abc1234"
```

**생략 가능한 항목**:
- Task 문서
- 구현 계획서
- Skills 분석
- 개별 회고
