## ✅ 3. 캐시 무효화 전략 (수용)

### 김감사 지적
> TD-003에서 "캐시 무효화 트리거 정의"만 언급, 구체적 전략 없음
> Write-Through Cache 제안

### 강철 대응

**✅ 수용: Write-Through Cache 전략 추가 예정**

**계획**:
TD-003 작업 시 다음 내용 포함 예정:

```javascript
/**
 * Write-Through Cache 전략
 * 데이터 쓰기 시 시트 + 캐시 동시 업데이트
 */
function registerTask(taskData) {
  // 1. 시트에 쓰기
  sheet.appendRow(taskData);

  // 2. 캐시 무효화
  CacheService.getScriptCache().remove('ALL_TASKS_CACHE');

  // 3. 새 데이터로 캐시 재생성
  getAllTasks(); // 내부에서 캐시 생성
}
```

**캐시 무효화 트리거 정의**:
1. 업무 등록 (registerTask)
2. 업무 수정 (updateTask)
3. 업무 삭제 (deleteTask)
4. 업무 상태 변경 (changeTaskStatus)
5. 업무 마감일 변경 (changeTaskDueDate)

**예상 효과**:
- 데이터 정합성 **100% 유지** (지연 시간 0초, 기존 최대 5분)
- 사용자 혼란 제거

**반영 시점**: TD-003 작업 시 (2026-03-10 전)

**판단 기준**:

| 상황 | 담당 팀 | 예시 |
|------|--------|------|
| **신규 개발** | 자비스 개발팀 | 기존 함수에 파라미터 추가 |
| **리팩토링** | 강철 AX팀 | 함수를 3개로 분리 |
| **회색 지대** | 강철 ↔ 자비스 협의 | 함수 수정 + 새 기능 추가 |

**구체적 예시**:
- 신규: `createTask(title, dueDate)` → `createTask(title, dueDate, priority)` (자비스)
- 리팩토링: 150줄 함수 → 3개 함수 분리 (AX)
- 회색: 기존 함수에 캐싱 추가 (성능 최적화 + 로직 변경) → 협의 필요

**예상 효과**:
- 팀 간 책임 소재 명확화 **100%**
- 중복 작업 제거
- 협업 효율 향상
