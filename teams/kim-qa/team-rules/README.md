## 📋 개요

**QA 전문 에이전트 팀**은 기존의 김감사 1인 QA 체제를 **4명의 전문가 팀**으로 확장하여 품질 검증 속도를 **57% 단축**하고 발견 가능한 버그를 **3배 증가**시키기 위해 구성되었습니다.

### 🎯 핵심 목표

1. **속도 개선**: 병렬 리뷰로 35분 → 15분 (57% 단축)
2. **품질 향상**: 기능 + 보안 + UX 전문가의 다각도 분석
3. **병목 해소**: 김감사 과부하 해소 → 팀 규칙 관리 및 회고에 집중
4. **전문성 강화**: 각 영역별 체크리스트 기반 체계적 검증

## 📝 QA 승인 기준

| 상태 | 조건 | 조치 |
|------|------|------|
| **✅ 승인** | Critical/High 이슈 **0개** | 즉시 배포 가능 |
| **⚠️ 조건부 승인** | Medium 이슈만 있음 | 배포 후 추적 관리 |
| **❌ 반려** | Critical/High 이슈 **1개 이상** | 수정 후 재검토 필요 |

### 우선순위 정의

- 🔴 **Critical**: 기능 중단, 데이터 손실, 보안 취약점
- 🟠 **High**: 주요 기능 심각한 영향, 사용자 경험 매우 나쁨
- 🟡 **Medium**: 기능 작동하나 불편, 일부 케이스 오작동
- 🟢 **Low**: 매우 작은 문제, 코드 품질 개선

## 🚀 시작하기

### QA 리뷰 요청 방법

1. **개발 완료 후** 김감사에게 QA 리뷰 요청
2. **변경 범위 명시**: Backend / Frontend / Full-Stack
3. **대상 파일 목록** 제공
4. **예상 소요 시간** 안내받기

### QA 리포트 작성 방법 (QA 전문가용)

1. `qa/templates/` 에서 해당 템플릿 복사
2. 파일명 규칙: `YYYY-MM-DD_feature_name_{functional|security|ux}_qa.md`
3. 체크리스트 기반으로 검증 진행
4. 시간 제한 엄수: Tester(15분), Security(10분), UX(10분)
5. 발견된 이슈는 우선순위별로 분류
6. 최종 판정: ✅ 승인 / ⚠️ 조건부 승인 / ❌ 반려

## ❓ FAQ

### Q1. Micro 작업 (< 30분)도 전체 QA 팀이 리뷰하나요?
**A**: 아니요. Fast-Track 프로세스 적용 → 테스터 1명 또는 김감사 자가 검토 (5분)

### Q2. QA 전문가들이 서로 소통하나요?
**A**: 아니요. 독립성 보장을 위해 **김감사에게만 질문/보고**합니다. 이유: 편향 없는 다각도 분석

### Q3. QA 시간 제한을 초과하면 어떻게 되나요?
**A**: 주요 이슈만 우선 리포트 제출합니다. (Tester: 10분 내, Security/UX: 7분 내)

### Q4. 긴급 핫픽스는 어떻게 진행되나요?
**A**: 전체 QA 팀 (3명) 동시 리뷰, 각 5분 병렬 처리 → Total 5분

### Q5. QA 템플릿은 어디에 있나요?
**A**: [qa/templates/](templates/) 폴더 참고
