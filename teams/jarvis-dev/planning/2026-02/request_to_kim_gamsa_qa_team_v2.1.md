# [요청] QA 팀 문서 체계 및 스킬 표기 방식 통일 건 (v2.1)

**수신**: 김감사 QA 팀장
**발신**: 자비스 (Jarvis) - 자비스 개발팀 PO
**날짜**: 2026-02-27
**제목**: AI 에이전트 팀 문서 표준화 및 전문 스킬(Skills) 기술 가이드 준수 요청
**회신 기한**: 2026-02-29 (금) 18:00까지

---

## 1. 개요

공도 업무 관리 시스템의 확장과 고도화에 따라, 각 에이전트 팀의 전문성을 명확히 하고 대외적인(팀장님 및 외부 감사) 신뢰도를 높이기 위해 **팀 소개 및 스킬셋 표기 방식을 표준화**하고자 합니다.

또한, **빠른 생산성과 아카이브**를 핵심 목표로 하여 실용적이고 측정 가능한 협업 체계를 구축합니다.

---

## 2. 주요 변경 및 요청 사항

### A. 팀원 소개 포맷 통일

자비스 팀의 [AI_AGENT_TEAM_OVERVIEW.md](../../docs/architecture/AI_AGENT_TEAM_OVERVIEW.md) 스타일에 맞춰, 김감사 QA 팀의 문서도 아래의 항목을 포함하도록 업데이트를 완료하였습니다. 앞으로 인원 충원이나 역할 변경 시 이 포맷을 유지해 주시기 바랍니다.

- **한줄 소개**: 에이전트의 정체성을 나타내는 페르소나 정의
- **핵심 역량 및 스택**: 보유 기술 및 담당 도메인
- **담당 업무 (R&R)**: 구체적인 책임 범위
- **장착된 전문 스킬 (Skills)**: 에이전트가 사용하는 실제 기술의 **공식 문서(URL)** 명시
- **협업 방식**: 타 에이전트와의 상호작용 방식

### B. 전문 스킬(Skills) 출처 명시 (중요)

에이전트의 환각(Hallucination)을 방지하고 기술적 근거를 확보하기 위해, 모든 스킬에는 반드시 **공식 Reference URL**을 첨부해야 합니다.

- 예: `OWASP Top 10`, `WCAG 2.1`, `ISTQB` 등 공식 표준 사이트 링크
- **검증 프로세스**: 신규 스킬 추가 시 알렉스 (Tech Lead)에게 사전 검증 요청
- **유효성 점검**: 연 1회 (매년 1월) 전체 스킬 링크 유효성 점검 실시

### C. 협업 팀(External Teams) 섹션 추가

팀 간의 유기적인 연결을 위해 본인의 팀 문서에서 상대 팀 문서를 상호 참조하는 섹션을 추가하였습니다.

### D. 문서 변경 이력(Changelog) 필수 관리 ⭐

**향후 모든 팀 문서는 Changelog를 필수로 관리**하여 변경 추적 및 아카이브를 보장합니다.

```markdown
## 📝 문서 변경 이력
| 버전 | 날짜 | 변경자 | 주요 변경 사항 |
|------|------|--------|----------------|
| v2.1 | 2026-02-27 | 자비스 팀 | 스킬 출처 URL 추가, 협업 팀 섹션 신설 |
| v2.0 | 2026-02-26 | 김감사 팀 | 병렬 QA 워크플로우 도입 |
```

**규칙**:
- 문서 수정 시 **반드시** Changelog 테이블에 1줄 추가
- 버전 번호 규칙: 주요 변경 시 v3.0, 부분 수정 시 v2.1, v2.2 등
- 변경자는 팀 단위 또는 개인 에이전트 명시

## 5. 협업 인터페이스 정의 (필수) ⭐

양 팀 간 명확한 산출물 교환 및 **아카이브 체계 확립**을 위해 아래 표준을 적용합니다.

### 5.1 자비스 개발 팀 → 김감사 QA 팀

| 항목 | 내용 |
|------|------|
| **산출물 형식** | `qa/qa_requests/{날짜}_{기능명}.md` |
| **명명 규칙** | 예: `qa/qa_requests/2026-02-27_slack_webhook.md` |
| **포함 정보** | - GitHub PR 링크<br>- 변경 파일 목록<br>- 테스트 시나리오<br>- 예상 리스크 |
| **전달 방식** | 1. 파일 생성<br>2. Slack `#qa-review` 채널 알림<br>3. 김감사에게 멘션 |

### 5.2 김감사 QA 팀 → 자비스 개발 팀

| 항목 | 내용 |
|------|------|
| **산출물 형식** | `qa/qa_reviews/{날짜}_{기능명}_report.md` |
| **명명 규칙** | 예: `qa/qa_reviews/2026-02-27_slack_webhook_report.md` |
| **포함 정보** | - 승인/반려 결정<br>- Critical/High/Medium 이슈 목록<br>- 수정 권고사항<br>- 재검수 필요 여부 |
| **전달 방식** | 1. 통합 리포트 생성<br>2. GitHub PR에 코멘트 작성<br>3. Slack `#dev-team` 채널에 알렉스 멘션 |

### 5.3 에스컬레이션 조건 (팀장님 승인 필요)

다음의 경우 양 팀장이 협의 후 팀장님(USER)께 승인 요청:

- 두 팀 간 표준 충돌 발생 시
- 새로운 스킬셋 카테고리 추가 시 (예: 성능 테스트 전문가 영입)
- 문서 구조 대폭 변경 시 (예: 새로운 워크플로우 섹션 추가)
- 주간 프로토타입 배포 실패 시 (목표 미달성)

## 6. 성과 측정 체계 (주간/월간/분기)

### 6.1 측정 철학
- **주간**: 빠른 피드백 및 즉각 조정 (1일 스프린트 → 주간 프로토타입)
- **월간**: 트렌드 파악 및 중기 계획 수립
- **분기**: 전략적 의사결정 및 ROI 산출 (휴먼 팀원 보고용)

### 6.2 품질 및 협업 지표

| 지표 | 주간 목표 | 월간 목표 | 분기 목표 | 측정 방법 |
|------|----------|----------|----------|-----------|
| **환각 방지율** | 0건 | 0건 | 0건 | 알렉스 코드 리뷰 로그 |
| **QA 통과율** | ≥ 80% | ≥ 85% | ≥ 90% | 승인 건수 / 전체 QA 건수 |
| **평균 QA 소요 시간** | ≤ 15분 | ≤ 13분 | ≤ 12분 | 병렬 QA 시간 추적 |
| **프로토타입 배포** | 1회 (금요일) | 4회 | 12회 | 주간 데모 완료 여부 |
| **문서 갱신율** | 100% | 100% | 100% | Changelog 업데이트 비율 |
| **휴먼 팀원 만족도** | ≥ 4.0/5.0 | ≥ 4.2/5.0 | ≥ 4.5/5.0 | 금요일 데모 후 설문 |

### 6.3 토큰 사용량 및 비용 측정 ⭐

**목적**:
- 회사 AI API 비용 측정 및 예산 수립
- 에이전트별 업무량 파악 및 작업 분배 최적화

| 측정 단위 | 측정 항목 | 목표 | 비고 |
|----------|----------|------|------|
| **일간** | 팀 전체 토큰 소비 | ≤ 100K tokens/day | 일일 작업량 모니터링 |
| **주간** | 에이전트별 토큰 소비 | 자비스 30K / 알렉스 50K / 에이다 20K / 클로이 15K / 벨라 10K | 업무 분배 밸런싱 |
| **월간** | 월 누적 비용 (예상) | ≤ $300/month | GPT-4 Turbo 기준 ($0.01/1K tokens) |
| **분기** | ROI 계산 | AI 비용 vs 절감 인건비 | 경영진 보고용 데이터 |

```

├── daily/
```
