# [기획 검토 요청] 주디(Judy) 모바일 앱 환경 구성안 (PWA & 단축어 도입)

**문서 상태**: 기획 초안 검토 요청 (Phase 24 사전 작업)
**작성일**: 2026-02-28
**작성자**: 자비스 (PO) / 벨라 (UX) / 알렉스 (Tech Lead)
**수신자**: 김감사 QA 팀장 및 관련 요원 (기능/보안/UX 사전 검토 요망)

## 1. 제안 배경 및 목적

현재 주디(AI 비서)와의 소통 및 데이터 관리는 **슬랙(Slack) 봇**과 **웹 워크스페이스(judy_workspace.html)**를 통해 이루어지고 있습니다. 

**문제점 (Pain Point)**:
- 사용자가 업무 중 주디에게 메모를 남기거나 지시를 내리기 위해 슬랙 앱을 켜고, 해당 워크스페이스를 찾아 들어가야 하는 **과도한 Depth(불편함)** 발생.
- 모바일(스마트폰) 환경에서 슬랙 앱 진입 없이 즉각적으로 시스템과 상호작용할 수 있는 '앱(App)과 같은 경험' 부재.

**해결 목적**:
슬랙 봇(알림 수신용) 뼈대는 유지하되, 스마트폰 홈 화면에서 **별도의 네이티브 앱처럼 동작하는 독립적인 진입점(Entry Point)**을 구축하여 사용자 생산성 및 접근성을 극대화합니다.

---

## 2. 모바일 환경 구성안 (3-Track 전략)

자비스 개발팀은 모바일 생산성 극대화를 위해 다음 3가지 방안을 동시/순차적으로 도입하고자 합니다.

### Track 1. PWA (Progressive Web App) 도입 (메인 대시보드 체류용)
웹 페이지를 스마트폰 홈 화면에 설치 가능한 '앱(App)' 형태로 변환하는 기술입니다.
- **적용 대상**: `judy_workspace.html` 기반의 메인 워크스페이스 시스템.
- **작동 방식**: 
  - 브라우저에 접속 시 "홈 화면에 추가" 프롬프트 제공 (Manifest.json 추가).
  - 스마트폰 배경화면의 주디 아이콘 클릭 시, 상하단 URL 표시줄 없는 Full-Screen(전체 화면) 독립 앱 형태로 실행.
- **기대 효과**: 슬랙을 거치지 않고 직접 칸반, 캘린더, AI 요약 툴에 접속 및 체류 가능.

### Track 2. iOS 단축어(Shortcuts) 아이콘 연동 (음성/텍스트 즉시 입력용)
아이폰(iOS) 사용자를 위한 바탕화면 즉시 메모 전송 기능입니다.
- **작동 방식**: 
  - 홈 화면의 "주디에게 지시" 단축어 아이콘 클릭 또는 Siri("Siri야, 주디에게 메모해 줘") 호출 시 네이티브 입력창 팝업.
  - 텍스트/음성 입력 완료 시 백그라운드에서 Google Apps Script(GAS) Webhook API로 다이렉트 POST 전송. (화면 전환 없음)
- **기대 효과**: 이동 중이거나 다른 앱 사용 중에도 1초 만에 시스템에 Task 전송 완료.

### Track 3. Android 위젯 (Macrodroid/Tasker) 기반 입력 (안드로이드 유저 대상)
안드로이드 사용자를 위한 홈 화면 위젯 연동입니다. (Track 2와 동일한 백엔드 API 재사용)
- **작동 방식**: 바탕화면에 투명 위젯이나 즉시 실행 팝업 버튼 배치. 누르면 텍스트 입력창이 뜨고 전송 시 GAS로 쏴줌.

## 3. 김감사 QA 팀 검토 요청 사항 (사전 리뷰)

본 기획을 실제 코드로 구현하기 앞서, QA 팀의 병렬 리뷰 체계에 입각하여 다음 항목들에 대한 리스크를 사전 검점해 주시기를 요청합니다.

### 🔍 테스터 (Functional QA)
- **GAS 백엔드 동시성 이슈**: iOS 단축어, 안드로이드 위젯, 슬랙봇 등 다중 채널에서 거의 동시에 데이터 전송 요청(Webhook WebApp 호출)이 들어올 경우 `LockService`가 이를 완벽히 방어/대기시킬 수 있는지 검토.
- **데이터 유실 방지 방안**: 모바일 데이터 통신망이 불안정한 상태에서 단축어를 통한 백그라운드 호출 시, 실패(Timeout) 처리 로직 및 재시도 메커니즘 설계 검토 필요 여부.

### 🛡️ 보안 감사관 (Security QA)
- **API 엔드포인트 노출 리스크**: 단축어 내부 URL이나 위젯 설정 파일에 GAS WebApp 엔드포인트가 평문으로 노출될 경우의 보안 위협(임의의 공격자가 URL을 알아내 스팸 Task를 생성할 가능성) 검토.
- **인증 토큰(Auth Token) 방안 검토**: 슬랙의 경우 자체 보안 토큰을 쓰지만, 모바일 단축어에서 쏠 때는 누가 보냈는지 검증할 `Bearer Token`이나 고유 사용자 식별 인증 시스템 설계안에 대한 피드백 요청.

### 🎨 UX 검증관 (UX/UI QA)
- **PWA 반응형 검증 준비**: `judy_workspace.html`이 PWA 전체 화면 모드로 띄워졌을 때, 화면 하단(Safe Area) 스와이프 제스처 취소나 노치 디자인과의 CSS 충돌(padding-top) 예상 리스크 검토.
- **사용자 피드백 루프 파괴**: 단축어/위젯을 통해 백그라운드로 보냈을 때, "성공적으로 저장되었습니다"라는 확실하고 직관적인 응답(피드백)을 모바일 네이티브 알림으로 띄우는 UX 플로우 필수 여부 확인 요망.
