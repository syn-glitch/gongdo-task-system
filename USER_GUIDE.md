# 공도 업무 관리 시스템 - 사용자 가이드

이 문서는 Slack 봇(주디)과 Google Sheets, Google Calendar를 연동하여 팀의 업무를 체계적으로 관리하는 시스템의 사용 방법을 설명합니다.
최근 추가된 **AI 에이전트 주디**와 1:1 대화(DM)를 통해 실시간으로 업무 현황 리포트를 받는 방법도 포함되어 있습니다.

---

## 🚀 워크플로우 (업무 등록부터 알림까지)

### 1단계: 새로운 업무 등록하기
업무 등록은 주로 Slack 내에서 언제 어디서나 간편하게 진행할 수 있습니다.

**[방법 A] 슬래시 명령어 사용 (기본)**
1. Slack 채팅창 아무 곳에서나 슬래시 명령어 **`/주디`** 를 입력하고 전송합니다.
2. `새 업무 등록` 모달 창이 팝업됩니다.

**[방법 B] 기존 메시지(댓글)를 바로 업무로 등록하기 (가장 추천하는 방법!)**
업무 요청이나 중요한 논의가 오간 채팅 메시지를 복사할 필요 없이, 즉시 주디봇의 업무로 변환할 수 있는 기능입니다.

1. 스레드나 채널에서 업무로 만들고 싶은 **원문 메시지(또는 댓글) 위로 마우스 커서**를 올립니다.
2. 우측에 뜨는 팝업 메뉴에서 **`[점 3개 (더보기)]`** 아이콘을 클릭합니다.
3. 가장 아래에 있는 **`앱에 연결`** -> **`더 많은 메시지 액션/명령 바로 가기...`**를 클릭합니다.
4. 검색창이 뜨면 **`이 메시지로 업무 등록`**을 검색하여 선택합니다.
   *(💡 팁: 한 번 사용하고 나면, 다음부터는 '점 3개' 메뉴 최상단에 바로 кнопки 버튼이 생겨서 광클 한 번으로 끝납니다!)*
5. 모달 창이 뜨며, 원본 작성자의 **실제 이름**과 텍스트 내용 전체가 '상세 내용' 칸에 완벽하게 자동 복사됩니다.

---

### [공통 입력 내용 및 담당자 할당]
모달 창(팝업창)이 열리면 아래 내용들을 채워주세요.

1. **프로젝트명 & 업무 제목 (필수):** 어떤 프로젝트 소속인지, 무슨 업무인지 간략히 적어주세요. (예: 공도 개발 / 앱 배포 준비)
2. **상세 내용 (선택):** 업무에 대한 자세한 설명이나 링크를 적어주세요. (방법 B를 쓰셨다면 자동 채워집니다!)
3. **마감일 (선택):** 달력을 띄워 마감일을 선택하시면, 시스템이 자동으로 담당자의 구글 캘린더에 당일 '종일 일정'으로 꽂아줍니다.
4. **담당자 배정 (선택 - ⭐️ 핵심 기능 ⭐️):**
   - 내가 해야 할 일이면 가만히 두시면 됩니다. (기본값: 본인)
   - **다른 사람에게 업무를 넘겨야(할당해야) 할 경우**, 드롭다운을 눌러 해당 팀원을 선택하세요!
5. **[등록 완료하기]** 버튼 클릭!

### 2단계: 자동 알림 및 캘린더 등록
버튼을 누르고 나면 주디봇이 백그라운드에서 눈썹 휘날리게 움직입니다. (약 1~2초 소요)
- **알림 1 (작성자):** 업무를 등록한 '나'에게 DM(개인 메시지)으로 "업무 등록/할당 완전 성공했어요!" 라고 카톡처럼 알려줍니다.
- **알림 2 (담당자):** 만약 내가 남에게 업무를 할당했다면? 주디봇이 **해당 팀원에게 직접 DM을 보내서** *"삐빅! 누가 당신에게 이런 업무를 배정했습니다!"* 하고 마감일과 함께 즉시 알려줍니다.
- **캘린더 & 시트:** 구글 시트(`Tasks`) 리스트에 한 줄이 예쁘게 추가되고, 구글 캘린더에도 자동으로 일정이 박힙니다.

### 3단계: 업무 상태 관리 및 알림 전파
Google Sheets의 `Tasks` 시트는 업무의 진행 상태를 중앙에서 관리하는 데이터베이스 역할을 합니다.
- **상태 변경:** 시트의 카테고리 셀 값을 **'대기'** 혹은 **'진행중'**으로 변경하면, 즉시 해당 프로젝트와 연결된 슬랙 채널(Projects 시트 기준)로 알림 메시지가 실시간 전송됩니다.
- 해당 알림 메시지 진행상황과 담당자, 캘린더 연동 여부(마감일 포함)등을 팀원들에게 공유합니다.

## 📂 데이터베이스(Google Sheets) 구조 및 관리법

이 시스템은 총 3개의 핵심 스프레드시트로 운영됩니다:

### 1. `Tasks` (업무 시트)
- **용도:** 실제 업무들이 등록되는 메인 시트입니다.
- **권장 사항:** 가급적 여기서 수동으로 데이터를 추가하기보다는 **Slack의 `/주디` 명령어를 사용**하는 것이 권장됩니다 (오타 방지 및 캘린더 연동 보장 등의 이점).
- **자동화 기능:** 상태(C열) 값이나 내용 등을 변경할 때마다 가장 우측의 **'최근 수정일'**이 자동으로 갱신되며, 마감일을 지우거나 상태를 '완료/보류'로 바꾸면 캘린더 스케줄에서도 자동으로 삭제됩니다.

### 2. `Projects` (프로젝트 관리 시트)
- **용도:** 각 프로젝트의 고유 식별 코드(Prefix)와 슬랙 알림을 보낼 연결 채널(URL 혹은 채널 ID 등)을 매핑하는 대조표입니다.
- 새로운 프로젝트가 시작되면 반드시 이 시트에 먼저 '프로젝트명', '코드(예: MKT, DEV 등)', '슬랙 채널(채널 ID)'을 등록해 두어야 슬랙과의 연동 및 자동 ID 부여가 정상 작동합니다.

### 3. `Users` (팀원 명부 시트)
- **용도:** 팀원의 이름과 정보 등의 메타 데이터를 보관합니다.
- 담당자 지정 등의 추후 디테일한 확장을 위해 사용됩니다.

## 🤖 AI 에이전트 (주디) 활용 가이드

주디는 단순 알림 봇을 넘어, 구글 시트에 저장된 팀의 전체 업무 데이터를 기반으로 질문에 대답할 수 있는 인공지능(Claude AI) 비서입니다.

### 💬 주디와 대화하는 방법 (실시간 리포트)
슬랙에서 주디(Judy Ops)를 클릭하여 **1:1 개인 메시지(DM) 창**을 열거나, 채널 내에서 **`@주디`** 를 멘션하여 자연어로 말을 걸어보세요.

**[질문 예시]**
- *"오늘 완수해야 할 주요 업무 보여줘"*
- *"이번 주 업무 중에 아직 처리 못한 업무 보여줘"*
- *"공도 개발 프로젝트 진행 상황 요약해 줘"*
- *"내(송용남)가 담당하는 대기 중인 업무가 뭐야?"*

**[작동 방식]**
1. 사용자가 질문을 던지면 주디가 구글 시트의 최신 데이터베이스를 실시간으로 읽어옵니다.
2. 강력한 언어 모델인 Claude AI가 5~10초 내에 데이터를 이해하고 분석합니다.
3. 분석된 결과를 바탕으로, 친절하고 읽기 쉬운 요약 답변을 슬랙 채팅으로 즉시 보내줍니다.

> 💡 **Tip:** 슬랙에 주디 봇에게 말을 거는 텍스트 입력창이 비활성화 되어 있다면, 슬랙 설정(`App Home > Messages Tab`)에서 채팅 기능이 켜져 있는지 확인하세요.

## 📝 주디 노트 & 데일리 메모장 활용 가이드 (New!)

이제 슬랙과 별도의 전용 웹앱을 통해 언제 어디서든 빠르게 아이디어를 쏟아내고, 그 내용을 구글 드라이브에 마크다운 문서로 안전하게 영구 보관할 수 있습니다. 작성된 메모들은 자동 분류되어 **`공도 주디봇 아카이브 / (이름) / YYYY-MM_업무일지.md`** 파일 1개에 일자/시간별로 누적 기록됩니다.

### 1. 슬랙 1:1 대화방(DM)에서 툭툭 던지기
바깥에 이동 중이거나 다른 탭에서 업무 도중 아이디어가 떠오르면, 슬랙 주디봇(Judy Ops)과 1:1 대화방을 열고 아무 말이나 텍스트로 전송해 보세요. (예: "오늘 오후 4시에 마케팅 회의 자료 준비할 것")
텍스트를 전송하는 즉시 주디봇이 백그라운드에서 구글 드라이브 마크다운 파일의 맨 아랫줄에 해당 내용을 꽂아 넣고, 성공 알림 메시지를 보냅니다.

### 2. 주디 노트(Web Notepad)에서 쾌적하게 작성하기
노트북이나 데스크탑 앞에 앉아 계실 때는 클릭이나 스크롤 같은 마찰이 아예 없는 극강의 미니멀리즘 웹앱, **주디 노트**를 크롬 즐겨찾기에 추가해 두고 활용하세요.

- **접속 방법:** 구글 앱스 스크립트 '웹 앱(Web App)' 으로 배포된 URL (`https://script.google.com/macros/s/.../exec`) 로 접속합니다. (슬랙 앱에 연동된 주소와 동일합니다)
- **사용법:**
  1. **[테마 및 사용자 설정]:** 상단의 ☀️/🌙 버튼을 통해 **다크모드/라이트모드**를 자유롭게 전환할 수 있습니다. 한 번 설정한 테마와 작성자 이름은 브라우저 환경에 영구 기억됩니다.
  2. **[과거 메모 조회 (강력한 기능!)]:** 사용자를 선택하면 구글 드라이브에 백업된 지난 문서들을 읽어와 **좌측 사이드바**에 폴더 구조 형태(월별 > 일자별) 로 보여줍니다. 특정 날짜를 클릭하면 우측 뷰어에 당시 작성한 아이디어가 한 눈에 표시됩니다.
  3. **[✨ AI 내용 요약]:** 과거 메모를 조회할 때 상단에 있는 버튼을 누르면, Claude AI가 1~3줄로 해당 내용을 짚어내어 즉시 오버레이로 보여줍니다.
  4. **[작성 및 자동 저장]:** 새로운 아이디어가 떠오를 땐 좌측의 `+ 새 메모` 를 눌러 모드를 변경합니다. 중앙 텍스트 에디터에 아무리 긴 내용이라도 편하게 적은 후 **`Ctrl + Enter`** 또는 **`Cmd + Enter`**를 누르면, 드라이브 아카이브(`.md`)에 저장되고 사이드바 트리에 일자가 즉시 업데이트됩니다.
