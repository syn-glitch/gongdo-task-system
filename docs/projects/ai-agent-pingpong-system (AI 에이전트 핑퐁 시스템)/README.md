## 📋 프로젝트 개요 (Project Overview)

### 🎯 프로젝트 목적 (Purpose)

공도 개발실의 AI 에이전트 팀(자비스 팀, 김감사 QA 팀) 간 협업 과정에서 발생하는 **수동 복사/붙여넣기 병목 현상**을 해결하기 위한 **무인 자동화 핑퐁 시스템** 구축 프로젝트입니다.

**핵심 가치**:
- ⏱️ **시간 절감**: 팀장님 개입 시간 57% 단축 (14분 → 6분)
- 🤖 **완전 자동화**: 구글 시트 기반 무인(Unattended) 워크플로우
- 📚 **RAG 연동**: GitHub 최신 룰북 실시간 동기화로 "살아있는 AI" 구현
- 🔄 **핑퐁 검수**: 자비스(개발) ↔ 김감사(QA) 자동 핑퐁으로 품질 보장

---

## 🚀 프로젝트 배경 (Background)

### AS-IS (현재 문제점)

```
1. 팀장님이 로컬(Antigravity)에서 요구사항 작성
2. 자비스 에이전트에게 복사/붙여넣기 → 개발 기획서 생성
3. 기획서를 복사하여 개발 에이전트에게 붙여넣기 → 코드 작성
4. 코드를 복사하여 김감사 에이전트에게 붙여넣기 → QA 검수
5. QA 결과를 복사하여 자비스에게 붙여넣기 → 수정
6. (4-5 반복 평균 3회)
7. 최종 승인 후 팀장님이 수동 배포

총 소요 시간: 평균 14분 (순수 복붙 작업 8분 + AI 응답 대기 6분)
```

**문제점**:
- 😓 팀장님이 모든 핑퐁 과정에 개입해야 함
- 🐛 복사 과정에서 컨텍스트 누락 위험
- ⏰ AI 응답 대기 중 다른 작업 불가능
- 📉 비효율적인 시간 사용

### TO-BE (목표 시스템)

```
1. 팀장님이 구글 시트(Agent_Tasks)에 요구사항 입력
2. [자동] 1분 단위 트리거가 감지 → 자비스 봇 자동 실행
3. [자동] 자비스 봇이 GitHub에서 최신 룰북 다운로드 → 개발 기획서 생성
4. [자동] 김감사 봇 자동 실행 → QA 검수
5. [자동] 에러 발견 시 자동으로 자비스에게 재요청 (최대 5회)
6. [자동] 최종 승인 시 슬랙으로 팀장님께 알림
7. 팀장님이 슬랙에서 결과 확인 후 배포 결정 (1분)

총 소요 시간: 평균 6분 (팀장님 개입 1분 + AI 자동 핑퐁 5분)
```

**개선 효과**:
- ✅ 팀장님 개입 시간 **87% 감소** (8분 → 1분)
- ✅ 전체 프로세스 시간 **57% 단축** (14분 → 6분)
- ✅ 컨텍스트 누락 위험 **제로**
- ✅ 팀장님은 최종 검토에만 집중 가능

## 🏆 핵심 혁신 포인트 (Key Innovations)

### 1. 🧠 RAG 실시간 동기화
**AS-IS**: 프롬프트에 룰북 하드코딩 → 룰북 업데이트 시 코드 수정 필요
**TO-BE**: GitHub에서 최신 룰북 실시간 다운로드 → 코드 수정 불필요

**예시 코드**:
```javascript
// AS-IS (하드코딩)
const prompt = "QA 프로세스: 1. Phase 0... 2. Phase 1..."; // 300줄 하드코딩

// TO-BE (RAG)
const qaRules = fetchGitHubRaw("qa/QA_PROCESS_V2.md"); // 실시간 동기화
const prompt = `QA 프로세스:\n${qaRules}`;
```

### 2. 🔄 무한루프 방지
**문제**: 자비스와 김감사가 서로 계속 에러를 주고받으며 무한 핑퐁
**해결**: 핑퐁 횟수 추적(L열) + 5회 초과 시 수동 에스컬레이션

### 3. 💰 비용 최적화
**예상 비용**: $90/월 (개발 모드 상시 가동)
**최적화 후**: $32/월 (개발/운영 모드 토글 + 배치 처리)
**절감률**: 64%

### 4. 🎯 스마트 알림
**AS-IS**: 모든 상태 변경 시 슬랙 알림 → 12개 알림 (피로도 ↑)
**TO-BE**: CRITICAL/HIGH만 발송 → 2개 알림 (83% 감소)

## 🎓 학습 목적 (Learning Objectives)

이 프로젝트는 다음과 같은 학습 자료로 활용됩니다:

### 1. AI 에이전트 팀을 위한 참고 자료
- **목적**: 향후 다른 프로젝트에서 AI가 이 문서를 RAG 소스로 활용
- **형식**: 구조화된 마크다운, 명확한 Before/After 비교, 코드 예시 포함

### 2. 인간 팀원 온보딩 자료
- **목적**: 신규 팀원이 공도 개발실의 AI 협업 프로세스 이해
- **내용**: 문제 정의 → 설계 → 협업 → 구현 → 회고 전 과정

### 3. 프로세스 표준화 템플릿
- **목적**: 유사한 자동화 프로젝트에 재사용 가능한 템플릿 제공
- **재사용 요소**: 폴더 구조, 문서 양식, 회고 템플릿

### 4. 의사결정 추적성 확보
- **목적**: "왜 이렇게 개발했는가?"에 대한 근거 보존
- **활용**: 감사 대응, 규제 준수, 아키텍처 리뷰
