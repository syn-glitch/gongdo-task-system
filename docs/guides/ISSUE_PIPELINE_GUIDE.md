# 이슈 자동화 파이프라인 — 사용자 가이드

## 개요

주디 워크스페이스에서 이슈를 제보하면 **AI 에이전트가 자동으로 분석하고 이해보고서를 작성**합니다.
사람이 개입하지 않아도 3단계가 자동으로 진행됩니다.

---

## 전체 흐름

```
팀원이 이슈 제보 (워크스페이스)
        │
        ▼
  1단계: GitHub Issue 자동 생성
        │
        ▼
  2단계: 김감사 QA 자동 분석 → 슬랙 DM (대표)
        │
        ▼
  3단계: 자비스 이해보고서 자동 작성 → 슬랙 DM (대표)
```

---

## 1단계: 이슈 제보하기

1. **주디 워크스페이스** 접속
2. 좌측 메뉴에서 **이슈 게시판** 클릭
3. **새 이슈 등록** 버튼 클릭
4. 제목, 본문, 카테고리를 입력
5. (선택) 스크린샷 첨부
6. **등록** 버튼 클릭

등록 즉시 GitHub Issue가 자동 생성되며, `from-workspace` 라벨이 붙습니다.

---

## 2단계: 김감사 QA 자동 분석

GitHub Issue가 생성되면 **김감사 QA팀**(AI)이 자동으로 분석을 시작합니다.

### 분석 내용
- **심각도 판정**: CRITICAL / MAJOR / MINOR
- **3축 분석**: 기능(40%) + 보안(30%) + UX(30%)
- **Overall Score**: 100점 만점
- **권장 조치**: 구체적 수정 방향

### 확인 방법
- **슬랙 DM**: 대표에게 자동 발송 (이슈 링크 포함)
- **GitHub Issue**: 코멘트에 QA 보고서가 자동 등록됨
- **라벨 변화**: `analyzing` → `qa-reviewed` (성공) 또는 `qa-failed` (실패)

### 소요 시간
약 **30초 ~ 1분**

---

## 3단계: 자비스 이해보고서

김감사 QA가 성공하면 **자비스 개발팀**(AI)이 자동으로 이해보고서를 작성합니다.

### 보고서 내용
- QA 보고서 핵심 요약
- 원인 추정 (코드 레벨)
- 영향 범위
- 수정 계획
- 예상 작업량 (소/중/대)
- 우선순위 의견

### 확인 방법
- **슬랙 DM**: 대표에게 자동 발송
- **GitHub Issue**: 코멘트에 이해보고서 등록
- **md 파일**: `reports/jarvis-response/` 폴더에 보고서 파일 자동 커밋
- **라벨**: `jarvis-reviewed` 추가

### 소요 시간
김감사 완료 후 약 **1분 ~ 2분**

---

## 슬랙 알림 요약

| 단계 | 발신자 | 수신자 | 내용 |
|------|--------|--------|------|
| 2단계 | 김감사 QA | 대표 | QA 분석 완료/실패 알림 + 이슈 링크 |
| 3단계 | 자비스 PO | 대표 | 이해보고서 완료/실패 + 이슈 링크 |

## FAQ

### Q: 이슈 등록 후 아무 반응이 없어요
- GitHub Actions 탭에서 워크플로우 실행 상태를 확인하세요
- `from-workspace` 라벨이 정상적으로 붙었는지 확인하세요
- 일일 분석 한도(20건/일)를 초과했을 수 있습니다

### Q: 김감사 분석은 됐는데 자비스가 안 돼요
- GitHub Actions 탭에서 "자비스 — 이슈 이해보고서" 워크플로우 확인
- 김감사 QA가 실패(`qa-failed` 라벨)한 경우 자비스는 실행되지 않습니다

### Q: 분석 결과가 이상해요
- AI 분석이므로 추정 기반입니다. 정확하지 않을 수 있습니다
- 코드 레벨 원인 추정은 참고용이며, 실제 디버깅은 별도로 필요합니다
