# 사용자 가이드 — 공도 업무 관리 자동화 파이프라인

- **작성자**: 꼼꼼이 (문서팀)

- **상태**: approved

## 1. 자동화 파이프라인이란?

워크스페이스에서 이슈를 제보하면, AI가 자동으로:
1. 이슈를 분석하고
2. 수정 계획을 세우고
3. 코드를 수정하고
4. 코드를 검수한 뒤
5. 대표님의 배포 승인을 받아 반영합니다.

**대표님이 해야 할 일은 딱 2가지**:
- 슬랙에서 **팀 배정 버튼** 클릭
- 슬랙에서 **배포 승인 버튼** 클릭

---

## 2. 이슈 제보 방법

### 워크스페이스에서 제보 (자동)
1. 주디 워크스페이스 접속
2. 우측 하단 **이슈 제보** 버튼 클릭
3. 카테고리, 심각도, 설명 입력
4. 제출 → GitHub 이슈 자동 생성 → 파이프라인 시작

### GitHub에서 직접 생성 (수동)
1. https://github.com/syn-glitch/gongdo-task-system/issues 접속
2. New Issue 클릭
3. 라벨에 `from-workspace` 추가 (이 라벨이 있어야 자동 분석 시작)
4. 제출

---

## 3. 슬랙 알림 안내

파이프라인이 진행되면 각 단계마다 슬랙 DM이 옵니다.

### 3.1 — 김감사 QA 분석 완료 알림

```
━━━━━━━━━━━━━━━━━━━━
🕵️ 김감사 QA 자동 분석
━━━━━━━━━━━━━━━━━━━━
🔢 이슈 #19: 전달된 업무 모달창 기능 없음
🏷️ 라벨: bug, from-workspace, severity-high
📋 상태: ✅ 분석 완료
💰 토큰: 1,234 in + 567 out | $0.0122 (₩17)

🔗 보고서 확인

📋 이슈 처리 담당팀을 선택해주세요:
[🤵 자비스 개발팀]  [🔧 강철 AX팀]     ← 버튼
```

**할 일**: 팀 배정 버튼 클릭
- **자비스**: 버그 수정, 신규 기능, UI 문제 → 자동 코드 수정 시작
- **강철**: 리팩토링, 성능 개선, 보안 강화 → 수동 처리

### 3.2 — 자비스 이해보고서 알림

```
━━━━━━━━━━━━━━━━━━━━
🤵 자비스 개발팀 이해보고서
━━━━━━━━━━━━━━━━━━━━
🔢 이슈 #19: 전달된 업무 모달창 기능 없음
📋 상태: ✅ 이해보고서 작성 완료
💰 토큰: 2,100 in + 890 out | $0.0197 (₩28)

확인 후 수정 작업 승인 부탁드립니다.

🔗 확인: (이슈 링크)
```

**할 일**: 이슈 링크 클릭하여 이해보고서 확인 (참고용, 별도 승인 불필요)

### 3.3 — 자비스 코드 수정 결과 알림

```
━━━━━━━━━━━━━━━━━━━━
🤵 자비스 자동 코드 수정 결과
━━━━━━━━━━━━━━━━━━━━
🔢 이슈 #19: 전달된 업무 모달창 기능 없음
📋 ✅ 코드 수정 완료 → PR 생성됨
💰 Claude Code Action (멀티턴) — 비용은 Anthropic 대시보드 참고
🔗 PR: (PR 링크)
⏳ 김감사 QA팀 재검토 진행 예정

🔗 이슈: (이슈 링크)
```

**할 일**: 없음 (자동으로 김감사 PR 리뷰가 시작됩니다)

### 3.4 — 김감사 PR 리뷰 + 배포 승인 버튼

```
🕵️ 김감사 PR 코드 리뷰 완료

이슈: #19 전달된 업무 모달창 기능 없음
PR: #20
판정: ✅ 승인
💰 토큰: 3,500 in + 1,200 out | $0.0285 (₩40)

[🚀 배포 승인]  [⏸️ 보류]     ← 버튼
```

**할 일**: 배포 버튼 클릭
- **🚀 배포 승인**: PR이 머지되고 이슈가 자동으로 닫힙니다
- **⏸️ 보류**: 나중에 GitHub에서 수동으로 처리합니다

## 4. 토큰 비용 확인

### 슬랙 메시지에서 확인
각 단계의 슬랙 알림에 토큰 사용량이 표시됩니다:

```
💰 토큰: 1,234 in + 567 out | $0.0122 (₩17)
```
- **in**: 입력 토큰 (이슈 내용 등 보낸 양)
- **out**: 출력 토큰 (AI가 생성한 양)
- **$**: 미국 달러 비용
- **₩**: 원화 환산 (x1,400)

### Anthropic 대시보드에서 전체 확인
- https://console.anthropic.com → Usage 탭
- 전체 API 사용량과 비용을 일별/월별로 확인 가능
- 특히 **Claude Code Action** (자비스 코드 수정)은 멀티턴이라 슬랙에 정확한 비용이 안 나오므로 대시보드에서 확인

### 이슈 1건당 예상 비용
- 단순 이슈: ~$0.10 (₩140)
- 복잡한 이슈 (코드 수정 포함): ~$0.50~1.00 (₩700~1,400)

## 5. 수동 트리거 방법

자동 트리거가 실패했거나 특정 단계만 다시 실행하고 싶을 때 사용합니다.

### GitHub Actions에서 수동 실행

1. https://github.com/syn-glitch/gongdo-task-system/actions 접속
2. 좌측에서 원하는 워크플로우 선택
3. **Run workflow** 버튼 클릭
4. 필요한 값 입력:
   - `issue_number`: 이슈 번호 (예: 19)
   - `pr_number`: PR 번호 (PR 리뷰 시만)
5. **Run workflow** 클릭

| 워크플로우 | 입력값 |
|----------|--------|
| 김감사 QA 분석 | 자동 (이슈 생성 시 실행) |
| 자비스 이해보고서 | `issue_number` |
| 자비스 코드 수정 | `issue_number` |
| 김감사 PR 리뷰 | `issue_number`, `pr_number` |

## 6. 판정 결과별 대응

| 김감사 판정 | 의미 | 대응 |
|-----------|------|------|
| ✅ 승인 | 코드 품질 양호, 보안 이슈 없음 | 배포 승인 버튼 클릭 |
| ⚠️ 조건부 승인 | 경미한 이슈 있으나 배포 가능 | 확인 후 승인 또는 보류 |
| ❌ 반려 | 심각한 문제 발견 | 보류 후 수동 확인 필요 |
| ⚠️ 리뷰 실패 | API 에러 등 | Anthropic 크레딧 확인 후 재실행 |

## 7. 자주 발생하는 문제 (FAQ)

### Q: 슬랙에 알림이 안 와요
**원인**: Slack Token 만료, Anthropic 크레딧 부족, 워크플로우 실패
**확인**: GitHub Actions 탭에서 실행 로그 확인
**조치**: 실패한 워크플로우 로그에서 에러 메시지 확인

### Q: "Your credit balance is too low" 에러
**원인**: Anthropic API 크레딧 소진
**조치**: https://console.anthropic.com → Plans & Billing → 크레딧 추가 구매

### Q: 팀 배정 버튼을 눌렀는데 반응이 없어요
**원인**: GAS 웹 앱 URL이 Slack Interactivity URL과 불일치
**확인**: GAS 실행 로그 (Apps Script → Executions)
**조치**: Slack App → Interactivity → Request URL이 최신 GAS 배포 URL인지 확인

### Q: PR이 머지가 안 돼요
**원인**: merge conflict 또는 브랜치 보호 규칙
**조치**: GitHub에서 수동으로 conflict 해결 후 머지

### Q: 자동 분석이 시작 안 돼요
**원인**: 이슈에 `from-workspace` 라벨이 없음
**조치**: 이슈에 수동으로 `from-workspace` 라벨 추가, 또는 Actions에서 수동 트리거

### Q: 같은 이슈가 중복 분석돼요
**원인**: 라벨 관리 오류
**확인**: 이슈에 `qa-reviewed` 라벨이 있으면 재실행 안 됨
**조치**: 재분석 원하면 `qa-reviewed` 라벨 제거 후 재트리거

## 8. 배포 후 추가 작업

PR 머지 = GitHub의 코드만 업데이트됩니다.
**실제 GAS 웹 앱에 반영하려면 추가 작업 필요**:

```
1. 로컬에서 git pull (최신 코드 받기)
2. clasp push (GAS에 코드 업로드)
3. Apps Script 에디터에서 배포 버전 업데이트 (수동)
```

> clasp push는 자동화되지 않습니다. 보안상 수동으로만 진행합니다.
