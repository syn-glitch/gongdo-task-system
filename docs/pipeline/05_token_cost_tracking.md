## 1. 요금 체계

### Claude Sonnet 4.5 요금

| 항목 | 가격 |
|------|------|
| Input tokens (입력) | **$3** / 1,000,000 tokens |
| Output tokens (출력) | **$15** / 1,000,000 tokens |

### 비용 계산 공식

```
비용(USD) = (입력 토큰 × 3 ÷ 1,000,000) + (출력 토큰 × 15 ÷ 1,000,000)
비용(KRW) = 비용(USD) × 1,400
```

### 예시

| 입력 토큰 | 출력 토큰 | USD | KRW |
|----------|----------|-----|-----|
| 1,000 | 500 | $0.0105 | ₩15 |
| 5,000 | 2,000 | $0.0450 | ₩63 |
| 10,000 | 5,000 | $0.1050 | ₩147 |

## 2. 각 워크플로우별 토큰 추적

### 2.1 — 김감사 QA 분석 (`kim-qa-issue-analysis.yml`)

**추적 방식**: Claude API 응답의 `usage` 필드에서 직접 추출

```python
usage = data.get("usage", {})
input_tokens = usage.get("input_tokens", 0)
output_tokens = usage.get("output_tokens", 0)
cost_usd = (input_tokens * 3 / 1_000_000) + (output_tokens * 15 / 1_000_000)
```

**슬랙 표시**:
```
💰 토큰: 1,234 in + 567 out | $0.0122 (₩17)
```

**GITHUB_OUTPUT으로 전달되는 값**:
- `input_tokens`
- `output_tokens`
- `cost_usd`

---

### 2.2 — 자비스 이해보고서 (`jarvis-issue-response.yml`)

**추적 방식**: 동일 (Claude API `usage` 필드)

**슬랙 표시**:
```
💰 토큰: 2,100 in + 890 out | $0.0197 (₩28)
```

---

### 2.3 — 김감사 PR 리뷰 (`kim-qa-pr-review.yml`)

**추적 방식**: 동일 (Claude API `usage` 필드)

**슬랙 표시** (Block Kit fields 내):
```
💰 토큰: 3,500 in + 1,200 out | $0.0285 (₩40)
```

---

### 2.4 — 자비스 코드 수정 (`jarvis-auto-fix.yml`)

**추적 방식**: ⚠️ **직접 추적 불가**

Claude Code Action은 멀티턴 대화(최대 40턴)로 진행되며,
각 턴의 토큰 사용량을 개별적으로 반환하지 않습니다.

**슬랙 표시**:
```
💰 Claude Code Action (멀티턴) — 비용은 Anthropic 대시보드 참고
```

**비용 확인 방법**: Anthropic 대시보드에서 해당 시간대의 사용량 확인

## 3. 이슈 1건 풀사이클 예상 비용

| 단계 | 유형 | 예상 입력 | 예상 출력 | 예상 비용 (USD) | 예상 비용 (KRW) |
|------|------|----------|----------|----------------|----------------|
| 1단계: QA 분석 | API 1회 | ~2,000 | ~1,000 | ~$0.02 | ~₩28 |
| 2단계: 이해보고서 | API 1회 | ~3,000 | ~1,500 | ~$0.03 | ~₩42 |
| 3단계: 코드 수정 | Code Action | 다회전 | 다회전 | ~$0.50~1.00 | ~₩700~1,400 |
| 4단계: PR 리뷰 | API 1회 | ~5,000 | ~1,500 | ~$0.04 | ~₩56 |
| **합계** | | | | **~$0.59~1.09** | **~₩826~1,526** |

> 코드 수정(3단계)이 전체 비용의 ~85%를 차지합니다.
> 단순 이슈는 코드 수정이 빠르게 끝나 ~$0.50 수준이고,
> 복잡한 이슈는 여러 파일 수정이 필요해 ~$1.00 이상일 수 있습니다.

## 5. Anthropic 대시보드에서 확인

### 접속 방법
1. https://console.anthropic.com 로그인
2. 좌측 메뉴 → **Usage** 탭

### 확인 가능한 정보
- 일별/월별 총 토큰 사용량
- 모델별 사용량 분류
- 총 비용 (USD)
- 잔여 크레딧

### 크레딧 충전
- Plans & Billing → Add Credits
- 신용카드로 충전 ($5 단위부터 가능)

## 6. 비용 절약 팁

### 6.1 불필요한 재실행 방지
- 라벨로 중복 실행 방지 (`qa-reviewed`, `jarvis-reviewed`, `fix-pr-created`)
- 일일 20회 제한으로 과도한 API 호출 차단

### 6.2 프롬프트 최적화
- `issue_triage_prompt.md` 등 시스템 프롬프트가 짧을수록 입력 토큰 절약
- PR diff 최대 15,000자 제한으로 과도한 입력 방지

### 6.3 코드 수정 턴 수 관리
- `jarvis-auto-fix.yml`의 `--max-turns 40` 값을 조절 가능
- 턴 수가 적으면 비용 절약, 많으면 복잡한 수정 가능

### 6.4 모델 선택
- 현재: Claude Sonnet (성능/비용 균형)
- 저렴한 대안: Claude Haiku (단순 분석에 적합, 비용 ~1/10)
- 단, 코드 수정 품질이 떨어질 수 있으므로 Sonnet 권장
