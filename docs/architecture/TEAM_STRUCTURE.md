## 🔁 핑퐁 시스템 (Pingpong System)

### 핑퐁이란?

**자비스 개발팀**과 **김감사 QA팀** 간의 **품질 보장을 위한 반복 검수 프로세스**

### 핑퐁 규칙

```
핑퐁 흐름:
1회차: 자비스 개발 → 김감사 QA → 반려 (3개 에러 발견)
2회차: 자비스 수정 → 김감사 QA → 반려 (1개 에러 발견)
3회차: 자비스 수정 → 김감사 QA → ✅ 승인!

규칙:
• 최대 핑퐁 횟수: 5회
• 5회 초과 시: 팀장님 수동 개입 (에스컬레이션)
• 승인 기준: CRITICAL 에러 0개 + Overall Score ≥ 80점
```

### 핑퐁 자동화 (진행중)

**프로젝트**: [AI 에이전트 핑퐁 시스템](../projects/ai-agent-pingpong-system%20(AI%20에이전트%20핑퐁%20시스템)/README.md)
- **목표**: 수동 복붙 병목 제거, 구글 시트 기반 무인 자동화
- **상태**: Phase 2 완료 (설계 및 QA 검토), Phase 4 구현 대기중
- **예상 효과**: 팀장님 개입 시간 87% 감소 (8분 → 1분)

```
Phase 1: 기획 (30분)
Phase 2: 개발 (가변)

  - 기능 테스트: 15분
  - 보안 감사: 10분
  - UX 검증: 10분
```

```
Phase 1: 기획 (30분)
Phase 2: 개발 (가변)
Phase 3: QA (15분) ← 3명이 동시 진행! ⚡
  - 테스터: 15분      ┐
  - 보안 감사관: 10분  ├→ 병렬 실행
  - UX 검증관: 10분    ┘
  - 김감사 통합: 10분
Phase 4: 배포 (5분)

Total QA 시간: 25분 (실제 대기 시간 15분)
```

**개선 효과**: QA 시간 **57% 단축** (35분 → 15분)

## 📋 팀 간 커뮤니케이션 규칙

### 공식 커뮤니케이션 채널

1. **개발 요청**: 팀장님 → 자비스 팀
   - **수단**: 구글 시트 (Agent_Tasks) 또는 슬랙 DM
   - **응답 시간**: 1시간 이내 (기획서 작성 시작)

2. **QA 요청**: 자비스 팀 → 김감사 팀
   - **수단**: 구글 시트 상태 변경 (`개발중` → `QA_대기`)
   - **응답 시간**: 2시간 이내 (QA 시작)

3. **QA 결과**: 김감사 팀 → 자비스 팀
   - **수단**: 통합 QA 보고서 + 슬랙 알림
   - **승인**: 바로 배포 준비
   - **반려**: 에러 리스트 + 수정 가이드 제공

4. **최종 보고**: 자비스 + 김감사 → 팀장님
   - **수단**: 슬랙 웹훅 알림 + 문서 링크
   - **내용**: 최종 QA 승인 완료 + 배포 준비 완료

## 🚀 향후 확장 계획

### Phase 1: 초기 구축 (완료)
```
자비스 개발 팀 (5명) + 김감사 QA 팀 (4명) = 총 9명
```

### Phase 2: AX 팀 추가 (완료)
```
자비스 개발 팀 (5명)
김감사 QA 팀 (4명)
+ 강철 AX 팀 (4명) ← 기술 부채, 리팩토링, 보안, 성능
= 총 13명
```

### Phase 3: 문서·인프라 팀 추가 (진행중)
```
자비스 개발 팀 (5명)
김감사 QA 팀 (4명)
강철 AX 팀 (4명)
+ 꼼꼼이 문서 팀 (2명) ← 문서 체계, 템플릿, 가이드
+ 벙커 인프라 팀 (6명 예정) ← 백오피스 자동화, 배포
= 총 21명
```

### Phase 4: AI 팩토리 (비전)
```
• 완전 자동화: 팀장님 요구사항 입력 → 자동 배포
• RAG 연동: GitHub 최신 룰북 실시간 동기화
• 무인 핑퐁: 자비스 ↔ 김감사 자동 핑퐁 (최대 5회)
• 슬랙 알림: 최종 결과만 팀장님께 보고
```
