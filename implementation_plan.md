# 업무 관리 시스템 - 구현 계획 (AI 최적화 버전)

## 목표 설명
비개발 직군 팀을 위해 **슬랙**, **구글 스프레드시트**, **Apps Script**, **구글 캘린더**, **Claude AI**를 활용한 통합 업무 관리 시스템을 구축합니다.

## 사용자 검토 필요
> [!TIP]
> **AI 에이전트 호환성 검토 결과**
> 현재 구조는 매우 훌륭합니다. 하지만 AI가 사람처럼 "이게 회의록 정리인지, 개발 업무인지" 판단하거나 "먼저 해야 할 일"을 찾기 위해서는 두 가지 정보가 더 있으면 완벽합니다.
> 1. **업무 유형 (Type)**: AI가 업무의 성격을 바로 파악하게 합니다. (예: 회의 -> 요약 리포트 작성, 개발 -> 코드 리뷰)
> 2. **선행 업무 (Dependency)**: "A를 끝내야 B를 할 수 있다"는 관계를 명시하면 AI가 스케줄링을 도울 수 있습니다.

---

## 1단계: 구글 스프레드시트 데이터베이스 설정 (최종)
AI가 문맥을 완벽히 이해할 수 있도록 컬럼을 보강했습니다.

### [확정] 1. `업무_DB` (Tasks) 시트 구조
| 열 | 컬럼명 (한글) | 설명 | AI 활용 포인트 |
| :--- | :--- | :--- | :--- |
| A | **업무 ID** | 예: `MKT-001` | AI가 인식하는 고유 키값 |
| B | **업무 유형** | `일반`, `회의`, `외근`, `개발`, `디자인` | **[NEW]** AI가 업무 성격에 따라 다른 액션을 취함 |
| C | **사용 여부** | `사용` : 진행 중 (목록 표시)<br>`중지` : 종료됨 (목록 숨김, 데이터 보존) | AI가 진행률 리포트 작성 시 사용 |
| D | **프로젝트** | 프로젝트명 | 프로젝트별 필터링 기준 |
| E | **업무 제목** | 업무의 핵심 요약 | AI 리포트의 헤드라인 |
| F | **상세 내용** | 구체적 지시사항 (줄바꿈 가능) | AI가 분석할 핵심 텍스트 데이터 |
| G | **담당자** | 실무자 이름 | 업무 배분 현황 분석 |
| H | **요청자** | 지시자 이름 | 커뮤니케이션 대상 파악 |
| I | **마감일** | YYYY-MM-DD | 일정 알림 및 자동 스케줄링 기준 |
| J | **선행 업무** | 먼저 끝내야 할 업무 ID (예: `DEV-001`) | **[NEW]** AI가 "이거 먼저 하세요"라고 제안 가능 |
| K | **우선순위** | `높음`, `중간`, `낮음` | 중요도 판단 |
| L | **슬랙 링크** | 스레드 URL | AI가 대화 내용을 긁어올 출처 |
| M | **캘린더 ID** | (시스템용) | 동기화 참조용 |
| N | **최근 수정일** | (자동) | 데이터 최신성 판단 |

### [확정] 2. `프로젝트_목록` (Projects) 시트
AI가 프로젝트 코드를 매핑하기 위한 참조 테이블입니다.
- **프로젝트명**
- **프로젝트 코드** (ID 접두사)
- **사용 여부**

### [확정] 3. `팀원_목록` (Users) 시트
AI가 사람을 인식하고 알림을 보낼 때 참조합니다.
- **이름**
- **슬랙 ID** (멤버 ID, 예: `U01234567`)
- **이메일**

---

## 검증 계획 - 1단계
1.  **시트 생성**: 위 구조대로 3개의 시트 생성.
2.  **데이터 입력**: 더미 데이터를 넣고 ID 생성 스크립트 작동 확인.
3.  **AI 시뮬레이션**: 입력된 데이터를 텍스트로 복사해서 Claude에게 "이 데이터를 보고 주간 업무 요약을 해줘"라고 시켜보고 잘 인식하는지 확인.
