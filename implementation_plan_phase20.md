# 20단계 Implementation Plan: 업무 라이프사이클 고도화 — 상태 관리, 시간 트래킹, 회고, 스마트 리마인더

> **최종 작성일**: 2026-02-24  
> **목표**: 업무의 등록 → 시작 → 완료까지의 전체 라이프사이클을 슬랙 모달에서 원스톱으로 관리하며, 시간 트래킹과 자기 회고를 통해 팀원의 성장과 AI 분석을 지원

---

## 📋 기능 분석 (Features)

사용자 요청에서 **4개의 핵심 기능**을 추출.

### 기능 1: 슬랙 모달 기반 업무 상태 관리
> *"구글 시트에서만 하던 상태 변경을 슬랙에서 한 번에"*

| 항목 | 내용 |
|---|---|
| **현재** | Tasks 시트를 직접 열어 상태 셀을 수동 변경 |
| **개선** | `/주디 내 업무` → 업무 목록에서 클릭 → 모달에서 상태 선택(시작/대기/보류/완료) → 시트 자동 반영 |
| **데이터** | Tasks 시트의 `상태` 컬럼을 슬랙 API로 직접 업데이트 |

### 기능 2: 업무 시간 트래킹 (Time Tracking)
> *"예상 시간 vs 실제 시간을 비교하여 자기 객관화"*

| 항목 | 내용 |
|---|---|
| **시작 시** | 업무 상태를 '진행중'으로 변경할 때 예상 소요 시간 입력 (예: 2시간) + 시작 타임스탬프 자동 기록 |
| **종료 시** | 업무 상태를 '완료'로 변경할 때 종료 타임스탬프 자동 기록 → 실제 소요 시간 자동 계산 |
| **분석** | `예상: 2시간 → 실제: 3.5시간 (75% 초과)` — AI가 패턴 분석 가능 |
| **데이터** | Tasks 시트에 `예상시간`, `시작시각`, `종료시각`, `실제시간` 컬럼 추가 |

### 기능 3: 업무 완료 회고 (Retrospective)
> *"왜 지연됐는지, 뭐가 잘 됐는지 자기 회고"*

| 항목 | 내용 |
|---|---|
| **트리거** | 업무 상태를 `완료`로 변경하는 순간, 회고 입력 모달 자동 팝업 |
| **입력 항목** | ① 잘된 점 (선택) ② 어려웠던 점 (선택) ③ 지연 사유 (시간 초과 시 자동 표시) ④ 한 줄 메모 (자유) |
| **활용** | AI가 팀원별 회고 데이터를 축적하여 주간/월간 성장 리포트 생성 |
| **데이터** | Tasks 시트에 `회고` 컬럼 추가 또는 별도 `Retrospectives` 시트 |

### 기능 4: 상태 기반 스마트 리마인더
> *"업무가 방치되지 않도록 주디가 알아서 찔러줌"*

| 항목 | 내용 |
|---|---|
| **자동 리마인더** | 업무가 `진행중`인데 N일째 변화 없음 → "이 업무 아직 진행 중이신가요?" DM 발송 |
| **마감 리마인더** | 마감일 D-1, D-Day에 자동 알림 |
| **직접 설정** | 업무 등록/상태 변경 시 "N일 후 리마인드" 직접 선택 가능 |
| **데이터** | Tasks 시트에 `리마인드일` 컬럼 추가 + GAS 타임 트리거로 매일 체크 |

## 🎬 사용자 시나리오 (User Scenarios)

### 시나리오 A: "마케팅 포스터 제작" 업무의 전체 라이프사이클

```
👤 이지은이 슬랙에서 업무를 등록합니다.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[1. 등록]
  /주디 업무등록
  → 모달: 프로젝트=MKT, 제목="홍보 포스터 제작", 마감일=2/28
  → Tasks 시트에 MKT-005 생성 (상태: 대기)

[2. 시작]  ← 🆕 NEW
  /주디 내 업무
  → "MKT-005 홍보 포스터 제작 [대기]" 클릭
  → 모달: [▶️ 시작하기] 버튼 클릭
  → "예상 소요 시간은?" → "3시간" 입력
  → 상태: 대기 → 진행중 | 시작시각: 2/25 14:00 자동 기록

[3. 리마인드]  ← 🆕 NEW
  (2일 후, 아직 진행중)
  🤖 주디 DM: "📌 MKT-005 '홍보 포스터 제작'이 2일째 진행 중이에요. 
               상태를 업데이트해주실래요?"
  → [계속 진행] [보류로 변경] [완료] 버튼

[4. 완료 + 회고]  ← 🆕 NEW  
  이지은이 [완료] 클릭
  → 종료시각: 2/27 17:30 자동 기록
  → 실제 소요: 약 5시간 (예상 3시간 대비 67% 초과)
  → 회고 모달 자동 팝업:
     ┌─────────────────────────────┐
     │ ⏱️ 예상: 3시간 → 실제: 5시간  │
     │                             │
     │ 잘된 점: [디자인 퀄리티 만족]   │
     │ 어려웠던 점: [소스 이미지 탐색]  │
     │ 지연 사유: [수정 요청 2회 발생]  │
     │ 한 줄 메모: [다음엔 사전 시안]   │
     └─────────────────────────────┘
  → Tasks 시트에 회고 내용 + 시간 데이터 저장

[5. AI 활용]
  🤖 주간 리포트: "이지은님 이번 주 완료 3건, 
     평균 예상 대비 실제 시간 +40%. 
     주요 지연 사유: 수정 요청(2건), 자료 부족(1건)"
```

### 리마인더 규칙 (자동 + 수동)

| 조건 | 리마인더 내용 | 방식 |
|---|---|---|
| 상태 `대기` + 등록 후 3일 경과 | "아직 시작 전이에요. 시작하시겠어요?" | 자동 |
| 상태 `진행중` + 2일째 변화 없음 | "진행 중 업무 상태를 업데이트해주세요" | 자동 |
| 마감일 D-1 | "내일 마감인 업무가 있어요" | 자동 |
| 마감일 D-Day | "오늘 마감입니다!" | 자동 |
| 사용자 직접 설정 | "N일 후 리마인드" | 수동 설정 |
